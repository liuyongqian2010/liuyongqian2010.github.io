定时任务crontab命令

定时任务crontab命令

1. 简介

  • 1.1 linux 系统是由 cron (crond) 这个系统服务来控制的。Linux 系统上面原本就有非常多的默认启动的计划性工作，由于使用者自己也可以设置计划任务，所以， Linux 系统也提供了使用者控制计划任务的命令 :crontab 命令
  • 1.2 当安装完成操作系统后，默认会安装此服务工具，并且会自动启动crond进程，crond进程会周期性检查是否有要执行的任务，如果有要执行的任务，则自动执行该任务。

2. 验证

  • 2.1 检查是否安装了crontab

    shell命令行：rpm -qa | grep crontab
目前版本 crontabs-1.11-6.20121102git.el7.noarch

  • 2.2 如果提示未安装请自行安装，yum install crontabs

  • 2.3 服务操作说明

    /sbin/service crond start         //启动服务
/sbin/service crond stop         //关闭服务
/sbin/service crond restart     //重启服务
/sbin/service crond reload         //重新载入配置
/sbin/service crond status         //启动服务

chkconfig –level 35 crond on       //加入开机自动启动

3. 系统任务调度

  • 3.1 系统周期性所要执行的工作，比如写缓存数据到硬盘、日志清理等。在/etc目录下有一个crontab文件，这个就是系统任务调度的配置文件。

  • 3.2 文件说明

    /etc/cron.deny      该文件中所列用户不允许使用crontab命令
/etc/cron.allow     该文件中所列用户允许使用crontab命令
/var/spool/cron/    所有用户crontab文件存放的目录,以用户名命名

4. 用户任务调度

  • 4.1 用户定期要执行的工作，比如用户数据备份、定时邮件提醒等。所有用户定义的crontab 文件都被保存在 /var/spool/cron目录中，其文件名与用户名一致。

  • 4.2 文件说明

    • 用户所建立的crontab文件中，每一行都代表一项任务，每行的每个字段代表一项设置，它的格式共分为六个字段，前五段是时间设定段，第六段是要执行的命令段

    • 格式如下：

      minute hour day month week command
其中：
minute：          表示分钟，可以是从0到59之间的任何整数。
hour：             表示小时，可以是从0到23之间的任何整数。
day：               表示日期，可以是从1到31之间的任何整数。
month：          表示月份，可以是从1到12之间的任何整数。
week：            表示星期几，可以是从0到7之间的任何整数，这里的0或7代表星期日。
command：        要执行的命令，可以是系统命令，也可以是自己编写的脚本文件。

    • 在以上各个字段中，还可以使用以下特殊字符：

      星号（*）：    代表所有可能的值，例如month字段如果是星号，则表示在满足其它字段的制约条件后每月都执行该命令操作。
逗号（,）：    可以用逗号隔开的值指定一个列表范围，例如，“1,2,5,7,8,9”
中杠（-）：    可以用整数之间的中杠表示一个整数范围，例如“2-6”表示“2,3,4,5,6”
正斜线（/）：    可以用正斜线指定时间的间隔频率，例如“0-23/2”表示每两小时执行一次。同时正斜线可以和星号一起使用，例如*/10，如果用在minute字段，表示每十分钟执行一次。

  • 4.3 环境变量EDITOR

    • 在考虑向cron进程提交一个crontab文件之前，首先要设置环境变量EDITOR。cron进程根据它来确定使用哪个编辑器编辑 crontab文件。多数LINUX用户都使用vi，那么需要编辑$HOME目录下的.bash_profile文件。

      cd /root
vi .bash_profile后添加如下：
EDITOR=vi
export EDITOR

5. crontab命令详解

  • 5.1 命令参数

    crontab -e              新建、编辑crontab文件
crontab -l               列出crontab文件
crontab -r               列出crontab文件

  • 5.2 实际应用

    • 5.2.1 由于实际工作中，在crontab中的命令均为shell脚本，而crontab本身没有环境给该shell运行，所以需要在shell脚本中最顶头添加如下命令：

      source /etc/profile
. ~/.bash_profile
#!/bin/bash
......

    • 5.2.2 备份数据库的实例（每天凌晨1:30进行备份）

      shell命令行执行：crontab -e 
添加如下命令：
30 1 * * *  /root/backup_database.sh

    • 5.2.3 backup_database.sh脚本如下所示：

      source /etc/profile        
. ~/.bash_profile        
#!/bin/bash

mysql_host="192.168.110.**"    
mysql_user="root"    
mysql_pwd="123456"    
backup_database="test_oa"
backup_date=$(date +%Y%m%d_%H%M%S)

mysqldump -h$mysql_host -u$mysql_user -p$mysql_pwd $backup_database > /root/test_oa_$backup_date.sql

完全备份与还原---mysqldump备份

完全备份与还原—mysqldump备份

1. 简介

  • 这篇笔记主要描述记录如何使用mysqldump命令来产生导出文件，和如何导入这些导出文件。

2. 备份数据

  • 2.1 备份一个数据库

    语法：shell模式下，mysqldump -u [username] -p [dbname] > backupName.sql

例如：（使用root用户备份名为test_oa的数据库）
mysqldump -u root -p test_oa > /root/bat_test_oa.sql
其中：
dbname参数表示需要备份的数据库的名称；
backupName.sql参数表示备份文件的名称，文件名前面可以加上一个绝对路径。

  • 2.2 备份数据库中的表

    语法：shell模式下，mysqldump -u [username] -p [dbname] [table1] [table2] ... > backupName.sql

例如：（使用root用户备份名为test_oa的数据库中的userinfo、workinfo两张表）
mysqldump -u root -p test_oa userinfo workinfo> /root/bat_test_oa_uw.sql
其中：table1和table2参数表示需要备份的表的名称，为空则整个数据库备份

  • 2.3 备份多个数据库

    语法：shell模式下，mysqldump -u [username] -p --databases [dbname1] [dbname2] > backupName.sql

其中：dbname1和dbname2参数表示需要备份的数据库的名称

  • 2.4 备份所有数据库

    语法：shell模式下，mysqldump -u [username] -p --all-databases > backupAll.sql

3. 还原数据

  • 3.1 还原一个数据库

    • 当使用上面的mysqldump语句导出文件的时候未使用到–databases或者–all-databases选项时，需要手动创建数据库

      语法：shell模式下，mysqladmin -u [username] -p create [dbname];
                 mysql -u [username] -p [dbname] < backupName.sql

例如：mysqladmin -uroot -p create test_oa;
     mysql -uroot -p test_oa < /root/bat_test_oa.sql
     其中：在导入文件时需要指定数据库名字

  • 3.2 还原多个数据库

    • 当使用上面的mysqldump语句导出文件的时候使用到了–databases或者–all-databases选项后，文件中已经含有了CREATE DATABASE和USE语句，所以不需要再指定一个数据库，可以直接读入所备份文件

      语法：shell模式下，mysql -u [username] -p < backupName.sql

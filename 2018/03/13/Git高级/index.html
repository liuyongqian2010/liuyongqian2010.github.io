Git高级

Git高级

分支管理

  • Git会串成一条时间线，这条时间线就是分支， 默认就是master
    HEAD并不是指向提交，而是指向master，指向最新版本

    v1  ==>  v2  ==> v3
                 |
                 master
                  |
                  HEAD
如果创建了新的分支，head将指向新的分支，如下
                        master
                          |
v1  ==>  v2  ==> v3  ==> v4
                          |
                        dev
                          |
                        HEAD

  • 基本命令

    • 查看分支：git branch（*表示你当前所处的branch是哪个）
    • 创建分支：git branch 分支名
    • 切换分支： git checkout 分支名
    • 创建并切换分支：git checkout -b 分支名
    • 合并分支：git merge 分支名 （需要切换到主分支或者依附分支再执行该命令）
    • 删除分支： git branch -d 分支名
    • 分支合并图：git log –graph –pretty=oneline –abbrev-commit

  • 解决冲突

    • 先在feature1 分支: a.txt中新增并提交‘AND simple’；
      后在master 分支: a.txt 中新增提交‘& simple’
      在master和分支上分别对a.txt做了不同修改之后，合并分支出现以下错误：

      git merge feature1
Auto-merging a.txt
CONFLICT (content): Merge conflict in a.txt
Automatic merge failed; fix conflicts and then commit the result.

    • 当Git无法自动合并分支时，就必须首先解决冲突，再提交，最后merge

  • 分支管理策略：

    • 1）master分支： 代码是稳定的，平时不能在上面干活的，仅仅是用来发布新版本用
    • 2）其他分支：是不稳定的，bug、feature可以创建新的分支进行开发，最后再合并到master即可

标签管理

  • 标签（tag），是版本库的一个快照；发布一个新版本时，通常先在版本库中打一个标签，无论在将来什么时候，取某个标签的版本即可

  • 基本命令

    git tag <tagname>创建标签
git tag查看所有标签
git tag -a <tagname> -m <desc>创建带有说明的标签
git show <tagname>查看标签的说明
git tag -d <tagname>删除标签

git push origin <tagname>推送一个本地标签
git push origin --tags推送全部未推送的本地标签

#如果tag已经推送到远程，要删除远程标签分2步：
1）删除本地tag:  git tag -d 本地标签名
2）git push origin :refs/tags/本地标签名

多人协作

  • 开源项目肯定都是很多人参与到社区中，通过协作完成
  • 多人协作常用操作
    • 克隆远程仓库：git clone xxxxxxx
    • 查看远程信息：git remove (-v)
    • 创建自己的分支：git branch xxxxxx
    • 推送分支：git push origin
    • 创建Pull Request

其他操作

  • git颜色设置

    Git控制台颜色设置
git config --global color.diff auto
git config --global color.status auto
git config --global color.branch auto

颜色设置关闭
git config --global color.status false

  • 忽略某些特殊文件不提交

    • 1）只需要在git项目的根目录下创建一个.gitignore
    • 2) 把需要提交的文件或者目录名称写到.gitignore
    • 3）如果你真正想提交的文件已经被包含在.gitignore里面了，那么就需要使用-f进行强制提交

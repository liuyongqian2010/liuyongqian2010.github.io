CentOS7安装Redis3.2遇到的问题

###到redis官网下载安装包，如”redis-3.2.8.tar.gz”；将下载的安装包放到 /opt下，分别使用：

  • tar -zvxf redis-3.2.8.tar.gz 解压安装包
  • cd redis-3.2.8 进入安装包
  • make 编译
    出现错误:

解决方法：安装gcc

  • yum -y install gcc automake autoconf libtool make 安装
  • make 运行
    出现错误：

解决方法：make改为make MALLOC=libc

  • make MALLOC=libc
  • make test 运行
    出现错误：

解决方法：升级tcl版本

  • yum install -y wget 安装wget
  • wget http://downloads.sourceforge.net/tcl/tcl8.6.1-src.tar.gz
  • sudo tar xzvf tcl8.6.1-src.tar.gz -C /usr/local/
  • cd /usr/local/tcl8.6.1/unix/
  • sudo ./configure
  • sudo make
  • sudo make install
  • 重新解压redis安装包并运行make进行编译
  • 进入src目录，./redis-server，如果出现虚线盒子界面表示运行成功
  • 可另起一个界面，运行 ./redis-cli来启动客户端
  • 可运行ps -ef|grep redis检测redis进程
  • 可运行netstat -Intp|grep 6379检测端口
  • 到此安装就结束了，我运用的远程控制软件是”SSH Secure Shell Client”

List类型操作

List类型操作

01

  • lpush key_name value1 value2…将一个或多个值插入到列表头部。 如果 key 不存在，一个空列表会被创建并执行lpush操作

    lpush mylist hello world

  • lpushx key_name value1 value2…将一个值插入到已存在的列表头部，列表不存在时操作无效

  • rpush key_name value1 value2…用于将一个或多个值插入到列表的尾部(最右边)
  • rpushx key_name value1 value2…用于将一个值插入到已存在的列表尾部(最右边)。如果列表不存在，操作无效
  • lpop key移出并获取列表的第一个元素
  • rpop key移出并获取列表的最后一个元素

02

  • lrange key_name start stop将一个值插入到已存在的列表头部，列表不存在时操作无效

  • lrem key_name count value根据参数 COUNT 的值，移除列表中与参数 VALUE 相等的元素

    • count的值可以是以下几种：

      • count > 0 : 从表头开始向表尾搜索，移除与value相等的元素，数量为value
      • count < 0 : 从表尾开始向表头搜索，移除与value相等的元素，数量为value的绝对值

      • count = 0 : 移除表中所有与value相等的值

        lrem mylist -2 hell0

  • rpoplpush用于移除列表的最后一个元素，并将该元素添加到另一个列表并返回

  • brpoplpush list1 another_list timeout从列表中弹出一个值，将弹出的元素插入到另外一个列表中并返回它； 如果列表没有元素会阻塞列表直到等待超时或发现可弹出元素为止

    brpoplpush msg reciver 500 #如在指定时间内没有任何元素被弹出，则返回一个 nil 和等待时长。 反之，返回一个含有两个元素的列表，第一个元素是被弹出元素的值，第二个元素是等待时长

  • blpop list1 list2…timeout移出并获取列表的第一个元素， 如果列表没有元素会阻塞列表直到等待超时或发现可弹出元素为止

    blpop list1 100          #如果列表为空，返回一个 nil 。 
                        否则，返回一个含有两个元素的列表，第一个元素是被弹出元素所属的 key ，
                        第二个元素是被弹出元素的值

  • brpop list1 list2…timeout移出并获取列表的最后一个元素， 如果列表没有元素会阻塞列表直到等待超时或发现可弹出元素为止

    brpop list 100           #假如在指定时间内没有任何元素被弹出，则返回一个 nil 和等待时长。         
                        反之，返回一个含有两个元素的列表，第一个元素是被弹出元素所属的 key ，
                            第二个元素是被弹出元素的值

03

  • lset key_name index value过索引来设置元素的值
  • ltrim key_name start stop 对一个列表进行修剪(trim)，让列表只保留指定区间内的元素

  • lindex kye_name index用于通过索引获取列表中的元素

    lpush mylist hello world
lindex mylist 1        #列表中下标为指定索引值的元素

  • linsert key_name before exist_value new_value用于在列表的元素前或者后插入元素。 当指定元素不存在于列表中时，不执行任何操作

    linsert mylist before hello 2018

  • llen key_name用于返回列表的长度

Hash类型操作

Hash类型操作

01

  • 1.hdel key_name filed_name用于删除哈希表 key 中的一个或多个指定字段，不存在的字段将被忽略

    hset myhash name liuyongqian
hdel myhash name         #result:被成功删除字段的数量

  • 2.hexists key_name filed_name用于查看哈希表的指定字段是否存在

    hexists myhash name      #result:如果哈希表含有给定字段，返回 1 ;
                    如果哈希表不含有给定字段，或 key 不存在，返回 0

  • 3.hget key_name field_name用于返回哈希表中指定字段的值

    hget myhash name         #result:返回给定字段的值。如果给定的字段或 key 不存在时，返回 nil

  • 4.hgetall key_name用于返回哈希表中，所有的字段和值

    hgetall myhash          #resutl:在返回值里，紧跟每个字段名(field name)之后是字段的值(value)

02

  • 5.hincrby key_name field_name value 用于为哈希表中的字段值加上指定增量值
    如果哈希表的 key 不存在，一个新的哈希表被创建并执行hincrby命令
    如果指定的字段不存在，那么在执行命令前，字段的值被初始化为 0

    hset myhash age 108
hincrby myhash age -54   #result:返回执行命令后哈希表中字段的值

  • 6.hincrbyfloat key_name field_name value用于为哈希表中的字段值加上指定浮点数增量值

    hincrbyfloat myhash age 1.5  #result:返回命令执行后的字段值

03

  • 7.hkeys key_name用于获取哈希表中的所有域（field）

    hkeys myhash             #result:包含哈希表中所有域（field）列表。 当 key 不存在时，返回一个空列表

  • 8.hvals key_name返回哈希表所有域(field)的值

    hvals myinfo          #result:一个包含哈希表中所有域(field)值的列表

  • 9.hlen key_name用于获取哈希表中字段的数量

    hlen myhash   #result:哈希表中字段的数量

04

  • 10.hmget key_name field1 field2 …用于返回哈希表中，一个或多个给定字段的值

    hmget myhash name age

  • 11.hmset key_name field1 value1 field2 value2 …用于同时将多个 field-value (字段-值)对设置到哈希表中,
    此命令会覆盖哈希表中已存在的字段

    hmset myinfo name liuyongqian age 108 work engineer

  • 12.hset key_name field_name用于为哈希表中的字段赋值
    如果哈希表不存在，一个新的哈希表被创建并进行 HSET 操作
    如果字段已经存在于哈希表中，旧值将被覆盖

    hset myonfo age 54

  • 13.hsetnx key_name field value用于为哈希表中不存在的的字段赋值
    如果哈希表不存在，一个新的哈希表被创建并进行 HSET 操作
    如果字段已经存在于哈希表中，操作无效
    如果 key 不存在，一个新哈希表被创建并执行 HSETNX 命令

    hsetnx myinfo limit  no

String类型操作

String类型操作

01

  • 1.set key value设置指定key的值

    set myname liuyongqian
set age 108

  • 2.get key获取指定key的值

    get myname

  • 3.getset key_name value用于设置指定key的值，并返回key的旧值，当key 没有旧值返回 nil

    getset myname lailai     #result:'liuyongqian'

  • 4.getrange key_name start end获取存储在指定key中字符串的子字符串

    set myname liuyongqian
getrange myname 0,2      #result:'liu'

  • 5.setrange key_name offset value用指定的字符串覆盖给定 key 所储存的字符串值，覆盖的位置从偏移量offset开始

    setrange job 5 manager
get job               #result:code-manager

  • 6.strlen key_name 用于获取指定 key 所储存的字符串值的长度。当 key 储存的不是字符串值时，返回一个错误

    strlen job             #result:12

02

  • 7.mset key1 value1 key2 value2 …用于同时设置一个或多个 key-value 对

    mset k1 hello k2 world     #result:OK

  • 8.setnx key_name value在指定的key不存在时，为key设置指定的值；若key存在，则设置失败，覆盖也失败

    exists job            #result:0
setnx job code-farmer #result:1

  • 9.msetnx key1 value1 key2 value2 …用于所有给定 key 都不存在时，同时设置一个或多个 key-value 对,若某个键已存在，则设置失败

  • 10.mget key1 key2 … 返回所有(一个或多个)给定 key 的值。 如果给定的 key 里面，有某个 key 不存在，那么这个 key 返回特殊值 nil 

    mget myname age         #result:'lailai''108'

03

  • 11.getbit keyname offset对key所储存的字符串值，获取指定偏移量上的位(bit),当偏移量offset比字符串值的长度大，或者key不存在时，返回0 

    set mynum a  #'a'的ASCII码是 97,转换为二进制是：1100001。offset的学名叫做“偏移” 。
             #二进制中的每一位就是offset值，比如在这里ffset 1等于'1' ,offset2等于'1',offset 7 等于'1' ，offset是从左往右计数的
getbit mynum 2           #result:1

  • 12.setbit key_name offset value 用于对key所储存的字符串值，设置或清除指定偏移量上的位(bit)

    #将mynum的值a变成b，也就是将01100001变成01100010（b的ASCII码是98），
#即将'a'中的offset 6从0变成1，将offset 7从1变成0
setbit mynum 6 1
setbit mynum 7 0
get mynum            #result:'b'

04

  • 13.setex key_name timeout value为指定的key设置值及其过期时间，单位为秒。如果 key 已经存在，将会替换旧的值

    setex mymemory 8 loveyou         #待8秒后mymemory的值为空

  • 14.psetex key_name timeout value以毫秒为单位设置 key 的生存时间

05

  • 15.incr key_name将key中储存的数字值增一
    如果key不存在，那么 key的值会先被初始化 0，然后再执行incr操作
    如果值包含错误的类型，或字符串类型的值不能表示为数字，那么返回一个错误

    set page 108
incr page
get page          #result:109

  • 16.incrby key_name value将key中储存的数字加上指定的增量值

    set rank 110
incrby rank 50
get rank          #result:160

  • 17.incrbyfloat key_name value为key中所储存的值加上指定的浮点数增量值

    set price 3.14
incrbyfloat price 0.01
get price         #result:3.15

  • 18.decr key_name将key中储存的数字值减一

  • 19.decrby key_name value将key所储存的值减去指定的减量值

  • 20.append key_name new_value 用于为指定的key追加值

    如果 key 已经存在并且是一个字符串，append命令将value追加到 key 原来的值的末尾
    如果key不存在，append就简单地将给定key设为 value ，就像执行set key value 一样

    exists myname       #result:0
append myname  liu
append myname  yongqian
get myname          #result:liuyongqian

MongoDB基本概念

MongoDB基本概念

分布式系统

  • 由多台计算机和通信的软件组件通过计算机网络连接（本地网络或广域网）组成
  • 建立在网络之上的软件系统。正是因为软件的特性，所以分布式系统具有高度的内聚性和透明性
  • 可以应用在不同的平台上如：Pc、工作站、局域网和广域网上等

MongoDB简介

  • 是由C++语言编写的，是一个基于分布式文件存储的开源数据库系统
  • 旨在为WEB应用提供可扩展的高性能数据存储解决方案
  • 数据存储为一个文档，数据结构由键值(key=>value)对组成。MongoDB 文档类似于 JSON 对象。字段值可以包含其他文档，数组及文档数组
  • Mongodb中的Map/reduce主要是用来对数据进行批量处理和聚合操作
  • Map和Reduce。Map函数调用emit(key,value)遍历集合中所有的记录，将key与value传给Reduce函数进行处理
  • Map函数和Reduce函数是使用Javascript编写的，并可以通过db.runCommand或mapreduce命令来执行MapReduce操作
  • GridFS是MongoDB中的一个内置功能，可以用于存放大量小文件

MongoDB安装与配置

  • 1.安装

    • 1.1下载安装包mongodb-win32-x86_64-2008plus-ssl-3.6.3-signed
    • 1.2安装时选择“Custom”自定义，安装路径为E:\mongodb

  • 2.创建数据库文件的存放位置
    因为启动mongodb服务之前需要创建数据库文件的存放文件夹，否则命令不会自动创建，而且不能启动成功

    • 2.1创建data目录，即E:\mongodb\data
    • 2.2创建db目录，即E:\mongodb\data\db

  • 3.启动mongodb服务

    • 3.1打开cmd命令行后cd E:\mongodb\bin目录下
    • 3.2接着输入启动服务命令：mongod –dbpath E:\mongodb\data\db

    • 3.3在浏览器输入http://localhost:27017查看，若显示“It looks like you are trying to access MongoDB over HTTP on the native driver port.”则表示连接成功

      mongod.exe　　-- MongoDB 的服务,通常用作启动服务用
mongo.exe　　-- MongoDB 的客户端,使用命令工具查询操作数据库

  • 4.配置本地windows mongodb服务
    在本地windows“服务”中，是没有配置上mongodb服务的，通过配置本地服务，可设置为开机自启动、手动启动或者命令行启动（net start/stop mongodb）

    • 4.1创建存放日志的文件夹在data下，并新建文件mongodb.log即E:\mongodb\data\logs\mongodb.log
    • 4.2创建配置文件在data下，即E:\mongodb\data\mongo.config

    • 4.3用记事本打开mongo.config，并输入：

      dbpath=E:\mongodb\data\db   #数据库路径
logpath=E:\mongodb\data\logs\mongodb.log #日志输出文件路径
logappend=true  #错误日志采用追加模式，追加到现有的日志文件，而不是重建一个新文件
journal=true    #启用日志文件，默认启用
quiet=true   #可以过滤掉一些无用日志信息，若需要调试使用请设置为false
port=27017   #端口号 默认为27017

    • 4.4用管理员身份打开cmd后，cd E:\mongodb\bin目录下，输入：mongod --config E:\mongodb\data\mongo.config --install --serviceName "mongodb"

    • 4.5安装后可查看本地服务（services.msc）
    • 4.6最好添加环境变量E:\mongodb\bin

  • 5.配置MongoDB插件

    • 5.1打开pycharm
    • 5.2File–>Settings–>Plugins–>Browse repositories–>Mongo–>Install
    • 5.3View->ToolButton

概念解析

  • 在mongodb中基本的概念是文档、集合、数据库

  • 数据库

    • 一个mongodb中可以建立多个数据库
    • MongoDB的默认数据库为”db”，该数据库存储在data目录中
    • MongoDB的单个实例可以容纳多个独立的数据库，每一个都有自己的集合和权限，不同的数据库也放置在不同的文件中
    • 有一些数据库名是保留的，可以直接访问这些有特殊作用的数据库
      • admin： 从权限的角度来看，这是”root”数据库。要是将一个用户添加到这个数据库，这个用户自动继承所有数据库的权限。一些特定的服务器端命令也只能从这个数据库运行，比如列出所有的数据库或者关闭服务器
      • local: 这个数据永远不会被复制，可以用来存储限于本地单台服务器的任意集合
      • config: 当Mongo用于分片设置时，config数据库在内部使用，用于保存分片的相关信息

  • 文档

    • 文档是一组有序的键值(key-value)对(即BSON)
    • 文档的键是字符串。除了少数例外情况，键可以使用任意UTF-8字符
    • MongoDB的文档不能有重复的键
    • 文档中的值不仅可以是在双引号里面的字符串，还可以是其他几种数据类型（甚至可以是整个嵌入的文档)
    • MongoDB区分类型和大小写
    • MongoDB 的文档不需要设置相同的字段，并且相同的字段不需要相同的数据类型，这与关系型数据库有很大的区别

  • 集合

    • 集合就是 MongoDB 文档组，类似于RDBMS中的表格
    • 集合存在于数据库中，集合没有固定的结构；当第一个文档插入时，集合就会被创建
    • 集合名不能以”system.”开头，这是为系统集合保留的前缀

  • 元数据

    • 数据库的信息是存储在集合中。它们使用了系统的命名空间：dbname.system.*
      • dbname.system.namespaces列出所有名字空间
      • dbname.system.indexes 列出所有索引
      • dbname.system.profile 包含数据库概要(profile)信息
      • dbname.system.users 列出所有可访问数据库的用户
      • dbname.local.sources 包含复制对端（slave）的服务器信息和状态

HTTP协议浅析

HTTP协议浅析

http概述

Q:http是干嘛的？

A:http是数据传输协议（超文本传输协议），用来沟通客户端和服务器的！

Q:怎么理解URI以及它的子集？

A:首先URI从其概念来说是统一资源标识符，它的作用就是在网络上唯一确定一个资源，就好比，在中国，身份证能唯一确定一个人一样！知道身份证号，就一定能确定一个人姓甚名谁一样！它有两个子集：URL(统一资源定位符)和URN(统一资源名)，首先不特别声明，我们所说的URI就是指URL，URL是跟资源其在网络上的位置有关！而URN是指资源跟其名字有关，URN是未来的趋势，不过貌似具体实施现在还在商讨中！所以短时间之内URN难以取代URL！

Q:什么是事务？

A:说白了事务就是“一次http链接（不包括tcp/ip连接，只包括一次http报文发送与接收）”的整个过程，由请求命令和响应结果组成！中间数据格式是http报文。我们平常打开一个网站，里面包括很多事务！如：请求网页文档、请求某个logo图片及请求某个视频等！

Q:方法指什么？

A:方法就是客户端向服务器发起的请求命令！常见方法有：get、post、delete、put、head！

Q:简单介绍一些报文！

A:首先报文是http协议一种纯文本的数据格式，分为请求报文和响应报文，两种报文都具有类似的结构，分别由三个部分构成：起始行、首部、主体，起始行描述报文干了什么！首部描述报文传输的具体细节！主体描述传输的实际内容！

Q:什么是TCP/IP？跟HTTP有什么关系？

A:tcp/ip是全世界的计算机和网络设备常用的层次化分组交换网络协议集！简单的说，http协议是一个应用层协议，位于tcp/ip协议的上一层，tcp/ip协议的主要作用就是过滤掉每个计算机的差异性，隐藏相关弱点，使得对于http协议来说提供的都是“相同的”接口！

Q:http协议有哪些版本？

A:

* http/0.9，这个版本有严重设计权限
* http/1.0，广泛使用
* http/1.0+ 非官方的http/1.0的扩展版本
* http/1.1 目前正在使用的版本，修复的相关设计缺陷，增加的相关特性
* http-NG 将来使用与否正在商讨中

URL

  • URI的分类
    • URI（统一资源标识符）由URL（统一资源定位符）和URN（统一资源名）两个子集构成
    • URL是从资源的位置来定义一个资源的，比如在“中国三东的一只小狗”和在“中国广东的一只小狗”就分别定义了两只不同的狗
    • URN是从资源的名字来定义的，比如小明和小李就分别定义了两个人
    • 目前的网络架构来说，大部分都是URL。URL的缺点就是如果资源的位置发生了改变，那么资源也就找不到了，而URN正是解决这个问题的，因为URN不受位置的限制，它只受名字的管理，因为对于URN来说，一个资源的名字是唯一的，所以无论资源移动到什么地方，都能通过名字定位到资源
    • 从某种意义上来说，URN是URI的未来趋势，但是URN的实现需要一个中间架构来满足这种位置到名字的映射，所以要完成从URL到URN的过度，显然是一个工程量的过程

  • URL的语法

    <scheme>://<user>:<password>@<host>:<port>/<path>;<params>?<query>#<frag>

scheme:方法描述了请求资源时用了什么协议，用“:”与url其它部分隔开；
user:用户名描述了访问是带的用户名；
password:密码描述了用户名后面可能跟的密码，用“:”跟用户名隔开；
host:主机描述了网站主机名或ip地址，如果前面有用户名和密码，用@分开；
post:服务器当前正在监听的端口，http默认为80，https默认为443;
path:路劲描述了资源在服务器上的位置，用‘/’跟前面部分隔开；
params:参数描述了请求需要附加的参数，用“;”与其他部分隔开；

Python的进程与线程

概念

  • 进程：独立（内存独立，cpu使用独立）的所有子任务的集合，启动进程开销大，进程之间很难共享数据和数据通信
  • 线程：进程中的每个子任务，不能独立存在，cpu执行的最小单位；依赖进程（内存共享）启动开销小，线程之间共享数据容易

  • 实现线程的两种形式

    • 函数方式实现：线程必须依赖函数实现，不能单独运行。单函数结束，线程结束。如果想让线程一致运行，就要想办法让程序不结束，比如在程序最后加一个input()或者死循环

资源共享引发的问题—锁

  • 由于并发的问题，需要加锁
  • 锁有锁定和未锁定两种状态，当一个线程要访问共享数据时，必须要先获得锁定，如果已经有别的线程获得锁定，那么就进入暂停状态，等别的线程把所释放后再进行操作

  • Conditon：更精确的控制锁，提供了四个方法，上锁-acquire()，等待-wait()，解锁-release()，唤醒-notify()

    import threading,time
sum=20  #总票数
count=0  #当前抢到票数
lock=threading.Lock()             #建锁
class qiangpiao(threading.Thread):
    def __init__(self,t_name):
        threading.Thread.__init__(self)
        self.name=t_name
    def run(self):
        global sum, count
        while True:
            lock.acquire()   #上锁
            if sum<=0:
                return
            else:
                sum-=1
                count+= 1
                print(self.name,'抢到了第{0}张票，剩余{1}张票'.format(count,sum))
            lock.release()    #释放
            time.sleep(1)
chuangkou=qiangpiao('窗口')
huangniu=qiangpiao('黄牛')
phone=qiangpiao('手机')
chuangkou.start()
huangniu.start()
phone.start()

python之XML

python之XML

初识XML

  • 定义
    • XML指可扩展标记语言（EXtensible Markup Language）,设计宗旨是传输数据，而不是显示数据,标签没有被预定义,需要自行定义标签,XML 被设计为具有自我描述性，被设计用来结构化、存储以及传输信息，是独立于软件和硬件的信息传输工具

  • 语法

    • XML 标签对大小写敏感。标签 \ 与标签 \<\letter> 是不同的
    • 在 XML 中，XML 的属性值必须加引号，如<note date="12/11/2007">

    • 在 XML 中，有 5 个预定义的实体引用

      &lt;    <    
&gt;    >    
&amp;    &    
&apos;    '    
&quot;    "
    • 没有什么规矩可以告诉我们什么时候该使用属性，而什么时候该使用元素。我的经验是在 HTML 中，属性用起来很便利，但是在 XML 中，您应该尽量避免使用属性。如果信息感觉起来很像数据，那么请使用元素
    • xml文件格式介绍：tag，即标签；attri，即属性；text，即文本字符串

  • 如下所示

    #文件名：countrys.xml
<?xml version="1.0"?>             <!--XML声明-->
<data id='0314' name="国家">       <!--根元素-->
    <country name="Liechtenstein">
        <rank>1</rank>
        <year>2008</year>
        <code>141100</code>
        <neighbor direction="E">Austria</neighbor>
    </country>

三种解析方式

  • 方式一：ElementTree(元素树)

    • ElementTree,类似一个轻量级的DOM解析XML文件的过程：

      • 1.导入ElementTree
      • 2.解析xml文件找到根节点：直接解析xml文件并获得根节点
      • 3.遍历根节点可以获得子节点，然后就可以根据需求拿到需要的字段

    • 演示

      import xml.etree.ElementTree as ET   #1.导入ElementTree
tree=ET.parse('countrys.xml')
root=tree.getroot()                  #2.直接加载解析xml文件并获得根节点
print('根节点:',root.tag,'根属性:',root.attrib,'根文本:',root.text)
for child in root:         #3.
    print('节点:',child.tag,'节点属性:',child.attrib,'节点文本:',child.text)
    for sub in child:
        print('子节点:',sub.tag,'子节点属性:',sub.attrib,'子节点文本:',sub.text)

  • 方式二：DOM(Document Object Model)

    • 一个 DOM 的解析器在解析一个XML文档时，一次性读取整个文档，把文档中所有元素保存在内存中的一个树结构里，之后你可以利用DOM 提供的不同的函数来读取或修改文档的内容和结构，也可以把修改过的内容写入xml文件。python中用xml.dom.minidom来解析xml文件

    • 演示

      from xml.dom.minidom import parse
import xml.dom.minidom
DOMTree=xml.dom.minidom.parse('countrys.xml') #使用minidom解析器打开xml文件
date=DOMTree.documentElement                  #获取根节点
if date.hasAttribute('name'):
    print('date.name=',date.getAttribute('name'))
countrys=date.getElementsByTagName('country') #获取所有子节点
for country in countrys:
    if country.hasAttribute('name'):
        print('country name:',country.getAttribute('name'))
        rank=country.getElementsByTagName('rank')[0]
        print('rank:',rank.childNodes[0].data)
        neighbor=country.getElementsByTagName('neighbor')[0]
        print('neighbor:',neighbor.childNodes[0].data)
        print('---------------------')

  • 方式三：SAX (simple API for XML )

    • SAX是一种基于事件驱动的API，利用SAX解析XML牵涉到两个部分：解析器和事件处理器。其中解析器负责读取XML文档，并向事件处理器发送事件，如元素开始跟元素结束事件；而事件处理器则负责对事件作出相应，对传递的XML数据进行处理。python中使用sax方式处理xml要先引入xml.sax中的parse函数，还有xml.sax.handler中的ContentHandler

    • 演示

      import xml.sax
#创建属于自己的读取xml文件的handler格式
class countryHandler(xml.sax.ContentHandler):
    def __init__(self):
        self.CurrentDate=''    #中间量，用来存储读取到的节点名字
        self.rank=''
        self.year=''
        self.neighber=''
    def startElement(self,tag,attributes):
        #startElement(tag, attrs)方法，遇到XML开始标签时调用，tag是标签的名字，attrs是标签的属性值字典
        self.CurrentDate=tag
        if tag=='country':
            name=attributes['name']
            print('-----------------------')
            print('country:',name)
    def characters(self,content):
        if self.CurrentDate=='rank':
            self.rank=content
        elif self.CurrentDate=='year':
            self.year = content
        elif self.CurrentDate=='neighbor':
            self.neighbor=content
    def endElement(self,tag):
        #endElement(name)方法,遇到XML结束标签时调用
        if self.CurrentDate=='rank':
            print('rank:',self.rank)
        elif self.CurrentDate=='year':
            print('year:',self.year)
        elif self.CurrentDate=='neighbor':
            print('neighbor:',self.neighbor)
        self.CurrentDate=''     #
if __name__=='__main__':
    parser=xml.sax.make_parser()     #创建一个xml读取器
    parser.setFeature(xml.sax.handler.feature_namespaces,0)
    handler=countryHandler()         #
    parser.setContentHandler(handler)
    parser.parse('countrys.xml')     #解析文档

Redis数据库入门

Redis数据库入门

NoSql(Not Only Sql)概念

  • NoSql是非关系型数据库的集合，如Redis、MongoDB、HBase…
  • NoSql产生背景，为解决三高而产生：数据库高并发大写；海量数据下的效率；高扩展和可用性
  • NoSQL优点：扩展方便，读写快速高效，数据类型灵活
  • NoSQL缺点：不支持sql语句

Redis简介

  • 由c语言编写，基于内存运行的并支持持久化，存储形式为key-value型
  • 单进程运行
    • redis服务程序是单进程运行，当多个客户端同时访问时，服务器的处理能力是会有一定的下降的。面对这种场景，我们可能会考虑在同一台服务器上开启多个Redis进程，Redis会对CPU存在竞争
  • Redis部署
    • 1.将redis安装包（此时最新版本是3.2.11）放入Linux服务器中
    • 2.解压redis，进入redis解压目录
    • 3.make
    • 4.make install，可指定安装目录，如make PREFIX="/opt/redis3" install
    • 5.之后安装目录中出现了bin目录，再将解压目录中的redis.conf放在与bin目录同级的目录下
    • 6.进入bin目录下，运行redis-server,出现层状方形小时运行成功
  • 在redis的bin目录下：
    • redis-server用来启动redis的工具，关闭该工具命令为shutdown
    • redis-benchmark检测redis在本机运行效率
    • redis-check-aof修复AOF持久化
    • redis-check-rdb修复RDB持久化文件
    • redis-cli命令行工具
  • Redis的多数据库（16个数据库：0-15 ）
    • Redis的多数据库在一定程度上是独立的,不支持自定义数据库名称，不能为每一个数据库设置独立密码
    • 切换数据库命令：select [dbindex]
    • 多数据库间移动数据：mv [key] [dbindex]
    • 清除数据库内数据：
      • 清空当前数据库数据：flushdb
      • 清空所有数据库数据：flushALL
    • 查看数据库内数据的数量：Dbsize
  • redis常用场景
    • 缓存
    • 排行榜类的应用
    • 实时分析系统

Redis参数配置

  • config命令：
    • 动态取出参数配置config get 参数名称，如config get port
    • 动态设置参数配置config set 参数名称
  • 配置分类：通用部分、快照、复制、安全、限制、追加、Lua脚本、慢日志、事件通知、高级配置、引入
  • Redis的基础配置（redis.conf）
    • notework:
      • bind: 指定要绑定的IP地址
      • protected-mode：询问是否开启保护模式；如不开启，bind会不起作用，密码也不起作用
      • port： 端口号，默认是6379，如果想开启多个Redis进程，需要指定不同的端口号
      • tcp-backlog：设置tcp的backlog的连接队列，即tcp的三次握手，未完成三次握手和已完成三次握手的队列总和
      • timeout：连接超时时间，默认是0
      • tcp-keepalive：检测keepalive的值，即健康检查，默认单位是秒。HA，默认值是0，表示不检测，建议设置为60秒
    • 通用配置：
      • daemonize：是否以后台进程的方式运行
      • pidfile: 指定pid存储的路径和文件，在正式使用时，一定会使用pidfile指定路径
    • 日志相关：
      • loglevel:四个级别
      • logfile：指定log日志文件位置,/dev/null
    • databases:
      • Redis启动后有多少个数据库
    • Redis限制部分配置[limits]
      • maxclients:客户端最大连接数，默认是10000
      • maxmemory：最大内存数，实际工作时，最大内存数一般都是50%左右的内存，一定不能超过70%
      • maxmemory-samples：样例数量
      • maxmemory-policy：设置内存的移除规则
        • volatile-lru：LRU算法进行移除，只针对设置了过期时间的值
        • allkeys-lru ：LRU算法
        • volatile-random：在过期集合中移除随机的key
        • allkeys-random：移除随机的key
        • volatile-ttl ：移除TTL最小的key，即数据的生命期
        • noeviction ：默认值，不进行移除

key操作

  • keys ：获得符合规则的键值列表，keys *
  • exists: 检测key是否存在，exists key名称
  • del: 单条删除key操作，del key名称
  • type:获取key对应的value值类型，type key名称
  • rename语句： rename oldKey newKey 不管新的key有没有都改名
  • renamenx: renamenx oldkey newkey 如果新的key存在，则不改名

数据类型

  • String类型
    • string是redis最基本的类型,可以包含任何数据。比如jpg图片或者序列化的对象
  • List类型
    • Redis 列表是简单的字符串列表，按照插入顺序排序。你可以添加一个元素到列表的头部（左边）或者尾部（右边）
  • Set类型
    • Redis的Set是string类型的无序集合。集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是O(1)
  • Hash类型
    • Redis hash 是一个键值(key=>value)对集合，是一个string类型的field和value的映射表，hash特别适合用于存储对象
  • Zset类型
    • Redis zset 和 set 一样也是string类型元素的集合,且不允许重复的成员。不同的是每个元素都会关联一个double类型的分数。redis正是通过分数来为集合中的成员进行从小到大的排序

分支管理

  • Git会串成一条时间线，这条时间线就是分支， 默认就是master
    HEAD并不是指向提交，而是指向master，指向最新版本

    v1  ==>  v2  ==> v3
                 |
                 master
                  |
                  HEAD
如果创建了新的分支，head将指向新的分支，如下
                        master
                          |
v1  ==>  v2  ==> v3  ==> v4
                          |
                        dev
                          |
                        HEAD

  • 基本命令

    • 查看分支：git branch（*表示你当前所处的branch是哪个）
    • 创建分支：git branch 分支名
    • 切换分支： git checkout 分支名
    • 创建并切换分支：git checkout -b 分支名
    • 合并分支：git merge 分支名 （需要切换到主分支或者依附分支再执行该命令）
    • 删除分支： git branch -d 分支名
    • 分支合并图：git log –graph –pretty=oneline –abbrev-commit

  • 解决冲突

    • 先在feature1 分支: a.txt中新增并提交‘AND simple’；
      后在master 分支: a.txt 中新增提交‘& simple’
      在master和分支上分别对a.txt做了不同修改之后，合并分支出现以下错误：

      git merge feature1
Auto-merging a.txt
CONFLICT (content): Merge conflict in a.txt
Automatic merge failed; fix conflicts and then commit the result.

    • 当Git无法自动合并分支时，就必须首先解决冲突，再提交，最后merge

  • 分支管理策略：

    • 1）master分支： 代码是稳定的，平时不能在上面干活的，仅仅是用来发布新版本用
    • 2）其他分支：是不稳定的，bug、feature可以创建新的分支进行开发，最后再合并到master即可

完全备份与还原---mysqldump备份

完全备份与还原—mysqldump备份

1. 简介

  • 这篇笔记主要描述记录如何使用mysqldump命令来产生导出文件，和如何导入这些导出文件。

2. 备份数据

  • 2.1 备份一个数据库

    语法：shell模式下，mysqldump -u [username] -p [dbname] > backupName.sql

例如：（使用root用户备份名为test_oa的数据库）
mysqldump -u root -p test_oa > /root/bat_test_oa.sql
其中：
dbname参数表示需要备份的数据库的名称；
backupName.sql参数表示备份文件的名称，文件名前面可以加上一个绝对路径。

  • 2.2 备份数据库中的表

    语法：shell模式下，mysqldump -u [username] -p [dbname] [table1] [table2] ... > backupName.sql

例如：（使用root用户备份名为test_oa的数据库中的userinfo、workinfo两张表）
mysqldump -u root -p test_oa userinfo workinfo> /root/bat_test_oa_uw.sql
其中：table1和table2参数表示需要备份的表的名称，为空则整个数据库备份

  • 2.3 备份多个数据库

    语法：shell模式下，mysqldump -u [username] -p --databases [dbname1] [dbname2] > backupName.sql

其中：dbname1和dbname2参数表示需要备份的数据库的名称

  • 2.4 备份所有数据库

    语法：shell模式下，mysqldump -u [username] -p --all-databases > backupAll.sql

3. 还原数据

  • 3.1 还原一个数据库

    • 当使用上面的mysqldump语句导出文件的时候未使用到–databases或者–all-databases选项时，需要手动创建数据库

      语法：shell模式下，mysqladmin -u [username] -p create [dbname];
                 mysql -u [username] -p [dbname] < backupName.sql

例如：mysqladmin -uroot -p create test_oa;
     mysql -uroot -p test_oa < /root/bat_test_oa.sql
     其中：在导入文件时需要指定数据库名字

  • 3.2 还原多个数据库

    • 当使用上面的mysqldump语句导出文件的时候使用到了–databases或者–all-databases选项后，文件中已经含有了CREATE DATABASE和USE语句，所以不需要再指定一个数据库，可以直接读入所备份文件

      语法：shell模式下，mysql -u [username] -p < backupName.sql

Appium的部署安装

Appium的部署安装

1. 简介

  • 1.1 Appium是一款开源测试自动化框架，可用于原生、混合和移动Web应用程序。它使用WebDriver协议驱动iOS，Android和Windows应用程序。重要的是，Appium是“跨平台”的：它允许您使用相同的API针对多个平台（iOS，Android，Windows）编写测试。
  • 1.2 工作原理：在Windows平台上，通过Python（ Python-client ）编写了一个Appium自动化脚本并执行，请求会首先到 AppiumForWindow.zip（Window下的Appium-Server），Appium-Server通过解析，然后调用Android模拟器或真机相关组件的接口从而驱动Android模拟器或真机来执行Appium自动化脚本。

2. 安装

  • 2.1 先要有一个基础的JAVA环境，即安装并配置好JDK，目前java version “1.8.0_201”。参考另一篇笔记《JDK的安装部署》

  • 2.2 安装Android SDK（安卓软件开发包），测试Android手机必然是要安装Android SDK的。

    • 下载地址：http://tools.android-studio.org/index.php/sdk/ ，目前下载installer_r24.4.1-windows.exe
    • 将安装目录新增至系统环境变量中，添加变量名：ANDROID_HOME 变量值: C:\andriodsdk

    • 双击SDK Manager，选装工具。

      必装：
Tools：Android SDK Tools、Android SDK Platform-tools、Android SDK Build-tools
镜像：本人安装的是 Android 5.1.1
Extras：Google USB Driver、Google Web Driver
Path环境变量中添加tools、platform-tools、build-tools目录。
测试：在CMD下执行adb devices（如果已经打开一个安卓虚拟机）查看结果

  • 2.3 安装node.js，由于Appium的server端是node.js编写的所以需要先安装node.js。安装node.js后，可以直接通过npm安装Appium。（注：npm类似于Python的pip，一个非官方包管理器）

    • 下载地址：https://nodejs.org/en/download/
    • 目前版本：node-v10.15.3

  • 2.4 安装appium服务端，

    • 第一种方式，由于网络问题建议通过cnpm安装appium

      通过npm命令安装cnpm
npm install -g cnpm --registry=https://registry.npm.taobao.org
通过cnpm安装appium
cnpm install -g appium --no-cache
安装成功后，通过appium -v 查看当前appium版本，直接输入appium则启动appium server端。

可以安装appium-doctor检查appium的所有需求环境，如下：
安装：npm install -g appium-doctor  
打开cmd，输入appium-doctor，检查环境是否OK，出现allchecks were successful，说明环境OK

    • 第二种方式，在Appium官方网站上下载操作系统相应的Appium版本

      • 下载地址：https://bitbucket.org/appium/appium.app/downloads/
      • 当前最新版本为 AppiumForWindows_1_4_16_1.zip ，注意这是一个 Windows 版本，如果你的电脑为MAC请下载 appium-1.5.3.dmg 。
      • 以 Windows 为例，将下载的 AppiumForWindows.zip 进行解压，点击 appium-installer.exe 进行安装。
      • 根据提示进行安装。最终在会桌面上生成 Appium图标 , 双击启动appium server。

  • 2.5 安装appium客户端，用于抓取app上定位信息。

    • 下载地址：https://github.com/appium/appium-desktop/releases/
    • 当前版本：Appium-windows-1.12.1
    • 双击安装，安装成功后桌面会有一个appium的快捷方式。

  • 2.6 选择一门编程语言，如python，使用Pycharm写代码；安装一个对应的Appium Client，推荐使用pip安装：pip install Appium-Python-Client

  • 2.7 选择一个测试运行环境，一个Android模拟器或者一个Android手机或者一台iPhone手机

使用JvisualVM远程监控JVM

使用JvisualVM远程监控JVM

1. 简介

  • 1.1 JvisualVM是jdk自带的可视化监控工。本篇简述在windows环境下监控远端的Linux系统的java进程
  • 1.2 windows环境下的jdk版本为：1.8.0_201
  • 1.3 Linux环境下的jdk版本也为：1.8.0_201

2. 远程Linux配置

  • 以tomcat为例，为tomcat 配置 jvm 启动参数。 在 tomcat 的 catalina.sh 中添加如下参数：

    JAVA_OPTS=
"-Djava.rmi.server.hostname=192.168.110.72
-Dcom.sun.management.jmxremote.port=8901 
-Dcom.sun.management.jmxremote.authenticate=false 
-Dcom.sun.management.jmxremote.ssl=false"
注： 
-Djava.rmi.server.hostname ：这个是配置 server 的 IP 
-Dcom.sun.management.jmxremote.port ：这个是配置远程 connection 的端口号，要确定此端口未被占用 
-Dcom.sun.management.jmxremote.ssl= 指定了 JMX 是否启用 ssl 
-Dcom.sun.management.jmxremote.authenticate= 指定了JMX 是否启用鉴权（需要用户名，密码鉴权）

3. 连接

  • 3.1 在window端找到{JAVA_HOME}/bin/jvisualvm.exe程序，并双击打开
  • 3.2 添加一个远程主机，在远程主机下，右键添加jmx连接（jmx：java management extensions ，即 Java 管理扩展）
  • 3.3 添加成功后，双击打开即可

  • 1.2 所需软件
    • Python 2.x 这是最基本的语言环境，RF框架是基于Python语言的，且WxPython只能支持到2.x的Python，所以此处不能下载最新的Python版本。

    • RIDE 是一个图形化界面， 用于创建、组织和运行测试用例的软件
    • Robot framework-selenium2library 可以看作RF版的selenium库。

2. 安装框架

  • 2.0 安装python
    • 环境变量配置的时候，需要将pip所在的目录也要加入环境变量，即python安装目录下的Scripts目录
    • 目前安装的是Python 2.7.16
  • 2.1 安装wxPython
    • 到wxPython的下载地址：https://sourceforge.net/projects/wxpython/files/wxPython/2.8.12.1/
    • 目前安装的是wxPython2.8-win64-unicode-2.8.12.1-py27
  • 2.2 安装robotframework
    • pip install robotframework
    • 目前安装的是robotframework-3.1.1
  • 2.3 安装robotframework-ride
    • pip install robotframework-ride
    • 目前安装的是robotframework-ride-1.7.3.1
    • 注意,可能会自动安装wxPython最新版，需要卸载最新版的wxPython：pip uninstall wxPython
  • 2.4 安装selenium2library扩展包
    • pip install robotframework-selenium2library
    • 目前安装的是robotframework-selenium2library-3.0.0

jenkins自动化部署–远程部署前端vue项目

1. Jenkins插件安装

  • 在系统管理—插件管理下，安装Publish over SSH插件

  • 在系统管理—系统设置下，配置SSH插件信息

    #只填写必要信息
SSH Server

    Name               front-web（随意一个名字即可）
    Hostname           192.168.110.**（需要部署的远程服务器IP）
    Username           root（登录远程服务器的用户名）
    Remote Directory      /usr/local/nginx/front-web（远程nginx服务器放置前端项目的目录）
    √ Use password authentication, or use a different key
    Passphrase / Password  （输入远程服务器登录密码）
    Port               22
    Timeout (ms)       300000

2. 构建项目

  • 2.1 新建任务（输入一个任务名称，如front-web，选择“构建一个自由风格的软件项目”）
  • 2.2 源码管理（选择“Git”，输入仓库URL和git账号）

  • 2.3 构建—执行shell，命令框中输入以下内容：

    node -v
npm -v

cd /var/lib/jenkins/workspace/front-web
rm -rvf dist
npm install
npm run build

  • 2.4 构建后操作—选择“Send build artifacts over SSH”，配置如下

    Transfer Set

    Source files        dist/css/,dist/fonts/,dist/img/,dist/js/,dist/*
    Remove prefix       dist
    Remote directory    #可不写，默认系统设置项/usr/local/nginx/front-web
    Exec command        #不需要命令，直接每次覆盖原来文件

3. Nginx配置

  • Nginx配置文件具体配置部分如下所示

    server {
        listen       80;
        server_name  localhost;

        location / {
            root   front-web;      #前端资源目录
            index  index.html index.htm;
        }

        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }

        location /api {
            proxy_pass http://192.168.110.**:8080/cmp;   #后台接口
        }
    }

      Transfer Set

    Source files        target/*.war
    Remove prefix         target/

    Exec command        sudo sh /usr/deploy_cmp.sh

      #!/bin/bash
export BUILD_ID=dontkillme
TOMCATTHREAD=`ps aux | grep 'tomcat' | grep -v grep`
if test -z "$TOMCATTHREAD"
then 
    echo "TOMCAT NOT START"
else  
    THREADLIST=(${TOMCATTHREAD// / })
    PID=${THREADLIST[1]}   
    kill -9 $PID   
    sleep 5s
fi  
    cd /usr/tomcat/webapps/   
    rm -f cmp.war  
    rm -rf cmp 
    cd /usr/tomcat/bin
    ./startup.sh
    sleep 5s

GCC升级

GCC升级

1. 选择升级版本

  • Linux系统默认安装的gcc版本均在4.9以下，一些软件比如NodeJS的编译需要比较高的版本
  • 选择需要升级的版本，http://mirror.koddos.net/gcc/releases/ ，当前选择版本为：gcc-7.4.0.tar.gz

2. 解压源码包

  • 将压缩包放在/usr/local下，进行解压：tar -zvxf gcc-7.4.0.tar.gz
  • 装备bzip2软件，之后的编译需要，执行：yum install bzip2

3. 开始编译安装

  • 下载供编译需要的依赖项

    cd /usr/local/gcc-7.4.0/ 后执行命令：
./contrib/download_prerequisites

  • 编译安装

    mkdir build  新建一个文件夹存放编译文件
cd build     后进行编译：
../configure --enable-languages=c,c++ --enable-checking=release  -disable-multilib  仅安装64位执行
make         生成Makefile文件，该过程可能会很耗时，起码90分钟
make install 之后执行安装命令

4. 解决运行程序gcc出现’GLIBCXX_3.4.21’ not found

  • 4.1 具体错误

    /usr/node-v10.15.3/out/Release/torque: /lib64/libstdc++.so.6: version `GLIBCXX_3.4.20' not found (required by /usr/node-v10.15.3/out/Release/torque)

  • 4.2 问题原因：升级gcc时，生成的动态库没有替换老版本 gcc 动态库导致

  • 4.3 解决方案：将gcc最新版本的动态库替换系统中老版本的动态库。

    • 查找编译gcc时生成的最新动态库 

      find / -name "libstdc++.so*"，
最新动态库为：/usr/local/gcc-7.4.0/build/stage1-x86_64-pc-linux-gnu/libstdc++-v3/src/.libs/libstdc++.so.6.0.24

    • 将找到的动态库libstdc++.so.6.0.24复制到/usr/lib64下

      cp /usr/local/gcc-7.4.0/build/stage1-x86_64-pc-linux-gnu/libstdc++-v3/src/.libs/libstdc++.so.6.0.24 /usr/lib64

    • 切换工作目录至/usr/lib64，删除原来的软连接，将默认库的软连接指向最新动态库。

      cd /usr/lib64
rm -rf libstdc++.so.6
ln -s libstdc++.so.6.0.24 libstdc++.so.6

5. 重启服务器，验证版本

  • 执行命令：gcc -v

4. 赋予 jenkins运行命令的权限
vi /etc/sudoers后增加如下命令：

## Allow root to run any commands anywhere 
root    ALL=(ALL)     ALL
jenkins    ALL=NOPASSWD: ALL

5. 使用 jenkins构建后台项目

  • 5.1 新建任务（输入一个任务名称，如testing001，选择“构建一个自由风格的软件项目”）
  • 5.2 源码管理（选择“Git”，输入仓库URL和git账号）

  • 5.3 修改maven配置文件

    cd /usr/share/maven/conf/
vi settings.xml后修改如下内容：（有两处）

<mirror>
      <id>mirrorId</id>
      <mirrorOf>repositoryId</mirrorOf>
      <name>Human Readable Name for this Mirror.</name>
      <url>http://my.repository.com/repo/path</url>
    </mirror>

<mirror>
      <id>mirrorId</id>
      <mirrorOf>*</mirrorOf>
      <name>Human Readable Name for this Mirror.</name>
      <url>http://192.168.110.67:8090/nexus/content/groups/public/</url>
    </mirror>

  • 5.4 构建

    • 调用顶层Maven目标，目标：clean package -Dmaven.test.skip=true

    • 执行shell，通过脚本进行部署，输入执行shell的命令，如：sudo sh /usr/deploy_cmp.sh。shell脚本可以参考如下所示：

      #!/bin/bash
export BUILD_ID=dontkillme
TOMCATTHREAD=`ps aux | grep 'tomcat-test' | grep -v grep`
if test -z "$TOMCATTHREAD"
then 
    echo "TOMCAT NOT START"
else  
    THREADLIST=(${TOMCATTHREAD// / })
    PID=${THREADLIST[1]}   
    kill -9 $PID   
    sleep 5s
fi  
    cd /usr/tomcat-test/webapps/   
    rm -f cmp.war  
    rm -rf cmp 
    cp /var/lib/jenkins/workspace/testing001/target/cmp.war /usr/tomcat-test/webapps/
    cd /usr/tomcat-test/bin
    ./startup.sh
    sleep 5s
  • 5.5 项目构建位置：/var/lib/jenkins/

HTTP协议浅析

HTTP协议浅析

http概述

Q:http是干嘛的？

A:http是数据传输协议（超文本传输协议），用来沟通客户端和服务器的！

Q:怎么理解URI以及它的子集？

A:首先URI从其概念来说是统一资源标识符，它的作用就是在网络上唯一确定一个资源，就好比，在中国，身份证能唯一确定一个人一样！知道身份证号，就一定能确定一个人姓甚名谁一样！它有两个子集：URL(统一资源定位符)和URN(统一资源名)，首先不特别声明，我们所说的URI就是指URL，URL是跟资源其在网络上的位置有关！而URN是指资源跟其名字有关，URN是未来的趋势，不过貌似具体实施现在还在商讨中！所以短时间之内URN难以取代URL！

Q:什么是事务？

A:说白了事务就是“一次http链接（不包括tcp/ip连接，只包括一次http报文发送与接收）”的整个过程，由请求命令和响应结果组成！中间数据格式是http报文。我们平常打开一个网站，里面包括很多事务！如：请求网页文档、请求某个logo图片及请求某个视频等！

Q:方法指什么？

A:方法就是客户端向服务器发起的请求命令！常见方法有：get、post、delete、put、head！

Q:简单介绍一些报文！

A:首先报文是http协议一种纯文本的数据格式，分为请求报文和响应报文，两种报文都具有类似的结构，分别由三个部分构成：起始行、首部、主体，起始行描述报文干了什么！首部描述报文传输的具体细节！主体描述传输的实际内容！

Q:什么是TCP/IP？跟HTTP有什么关系？

A:tcp/ip是全世界的计算机和网络设备常用的层次化分组交换网络协议集！简单的说，http协议是一个应用层协议，位于tcp/ip协议的上一层，tcp/ip协议的主要作用就是过滤掉每个计算机的差异性，隐藏相关弱点，使得对于http协议来说提供的都是“相同的”接口！

Q:http协议有哪些版本？

A:

* http/0.9，这个版本有严重设计权限
* http/1.0，广泛使用
* http/1.0+ 非官方的http/1.0的扩展版本
* http/1.1 目前正在使用的版本，修复的相关设计缺陷，增加的相关特性
* http-NG 将来使用与否正在商讨中

URL

  • URI的分类
    • URI（统一资源标识符）由URL（统一资源定位符）和URN（统一资源名）两个子集构成
    • URL是从资源的位置来定义一个资源的，比如在“中国三东的一只小狗”和在“中国广东的一只小狗”就分别定义了两只不同的狗
    • URN是从资源的名字来定义的，比如小明和小李就分别定义了两个人
    • 目前的网络架构来说，大部分都是URL。URL的缺点就是如果资源的位置发生了改变，那么资源也就找不到了，而URN正是解决这个问题的，因为URN不受位置的限制，它只受名字的管理，因为对于URN来说，一个资源的名字是唯一的，所以无论资源移动到什么地方，都能通过名字定位到资源
    • 从某种意义上来说，URN是URI的未来趋势，但是URN的实现需要一个中间架构来满足这种位置到名字的映射，所以要完成从URL到URN的过度，显然是一个工程量的过程

  • URL的语法

    <scheme>://<user>:<password>@<host>:<port>/<path>;<params>?<query>#<frag>

scheme:方法描述了请求资源时用了什么协议，用“:”与url其它部分隔开；
user:用户名描述了访问是带的用户名；
password:密码描述了用户名后面可能跟的密码，用“:”跟用户名隔开；
host:主机描述了网站主机名或ip地址，如果前面有用户名和密码，用@分开；
post:服务器当前正在监听的端口，http默认为80，https默认为443;
path:路劲描述了资源在服务器上的位置，用‘/’跟前面部分隔开；
params:参数描述了请求需要附加的参数，用“;”与其他部分隔开；
query:查询是用来激活服务器程序去执行某些操作，比如查询数据库等，用“?”与其余部分隔开；
frag:片段只在客户端使用，不发送到服务器端；

Python的进程与线程

Python的进程与线程

概念

  • 进程：独立（内存独立，cpu使用独立）的所有子任务的集合，启动进程开销大，进程之间很难共享数据和数据通信
  • 线程：进程中的每个子任务，不能独立存在，cpu执行的最小单位；依赖进程（内存共享）启动开销小，线程之间共享数据容易

  • 实现线程的两种形式

    • 函数方式实现：线程必须依赖函数实现，不能单独运行。单函数结束，线程结束。如果想让线程一致运行，就要想办法让程序不结束，比如在程序最后加一个input()或者死循环

资源共享引发的问题—锁

  • 由于并发的问题，需要加锁
  • 锁有锁定和未锁定两种状态，当一个线程要访问共享数据时，必须要先获得锁定，如果已经有别的线程获得锁定，那么就进入暂停状态，等别的线程把所释放后再进行操作

  • Conditon：更精确的控制锁，提供了四个方法，上锁-acquire()，等待-wait()，解锁-release()，唤醒-notify()

    import threading,time
sum=20  #总票数
count=0  #当前抢到票数
lock=threading.Lock()             #建锁
class qiangpiao(threading.Thread):
    def __init__(self,t_name):
        threading.Thread.__init__(self)
        self.name=t_name
    def run(self):
        global sum, count
        while True:
            lock.acquire()   #上锁
            if sum<=0:
                return
            else:
                sum-=1
                count+= 1
                print(self.name,'抢到了第{0}张票，剩余{1}张票'.format(count,sum))
            lock.release()    #释放
            time.sleep(1)
chuangkou=qiangpiao('窗口')
huangniu=qiangpiao('黄牛')
phone=qiangpiao('手机')
chuangkou.start()
huangniu.start()
phone.start()

python之XML

python之XML

初识XML

  • 定义
    • XML指可扩展标记语言（EXtensible Markup Language）,设计宗旨是传输数据，而不是显示数据,标签没有被预定义,需要自行定义标签,XML 被设计为具有自我描述性，被设计用来结构化、存储以及传输信息，是独立于软件和硬件的信息传输工具

  • 语法

    • XML 标签对大小写敏感。标签 \ 与标签 \<\letter> 是不同的
    • 在 XML 中，XML 的属性值必须加引号，如<note date="12/11/2007">

    • 在 XML 中，有 5 个预定义的实体引用

      &lt;    <    
&gt;    >    
&amp;    &    
&apos;    '    
&quot;    "
    • 没有什么规矩可以告诉我们什么时候该使用属性，而什么时候该使用元素。我的经验是在 HTML 中，属性用起来很便利，但是在 XML 中，您应该尽量避免使用属性。如果信息感觉起来很像数据，那么请使用元素
    • xml文件格式介绍：tag，即标签；attri，即属性；text，即文本字符串

  • 如下所示

    #文件名：countrys.xml
<?xml version="1.0"?>             <!--XML声明-->
<data id='0314' name="国家">       <!--根元素-->
    <country name="Liechtenstein">
        <rank>1</rank>

三种解析方式

  • 方式一：ElementTree(元素树)

    • ElementTree,类似一个轻量级的DOM解析XML文件的过程：

      • 1.导入ElementTree
      • 2.解析xml文件找到根节点：直接解析xml文件并获得根节点
      • 3.遍历根节点可以获得子节点，然后就可以根据需求拿到需要的字段

    • 演示

      import xml.etree.ElementTree as ET   #1.导入ElementTree
tree=ET.parse('countrys.xml')
root=tree.getroot()                  #2.直接加载解析xml文件并获得根节点
print('根节点:',root.tag,'根属性:',root.attrib,'根文本:',root.text)
for child in root:         #3.
    print('节点:',child.tag,'节点属性:',child.attrib,'节点文本:',child.text)
    for sub in child:
        print('子节点:',sub.tag,'子节点属性:',sub.attrib,'子节点文本:',sub.text)

  • 方式二：DOM(Document Object Model)

    • 一个 DOM 的解析器在解析一个XML文档时，一次性读取整个文档，把文档中所有元素保存在内存中的一个树结构里，之后你可以利用DOM 提供的不同的函数来读取或修改文档的内容和结构，也可以把修改过的内容写入xml文件。python中用xml.dom.minidom来解析xml文件

    • 演示

      from xml.dom.minidom import parse
import xml.dom.minidom
DOMTree=xml.dom.minidom.parse('countrys.xml') #使用minidom解析器打开xml文件
date=DOMTree.documentElement                  #获取根节点
if date.hasAttribute('name'):
    print('date.name=',date.getAttribute('name'))
countrys=date.getElementsByTagName('country') #获取所有子节点
for country in countrys:
    if country.hasAttribute('name'):
        print('country name:',country.getAttribute('name'))
        rank=country.getElementsByTagName('rank')[0]
        print('rank:',rank.childNodes[0].data)
        neighbor=country.getElementsByTagName('neighbor')[0]
        print('neighbor:',neighbor.childNodes[0].data)
        print('---------------------')

  • 方式三：SAX (simple API for XML )

    • SAX是一种基于事件驱动的API，利用SAX解析XML牵涉到两个部分：解析器和事件处理器。其中解析器负责读取XML文档，并向事件处理器发送事件，如元素开始跟元素结束事件；而事件处理器则负责对事件作出相应，对传递的XML数据进行处理。python中使用sax方式处理xml要先引入xml.sax中的parse函数，还有xml.sax.handler中的ContentHandler

    • 演示

      import xml.sax
#创建属于自己的读取xml文件的handler格式
class countryHandler(xml.sax.ContentHandler):
    def __init__(self):
        self.CurrentDate=''    #中间量，用来存储读取到的节点名字
        self.rank=''
        self.year=''
        self.neighber=''
    def startElement(self,tag,attributes):
        #startElement(tag, attrs)方法，遇到XML开始标签时调用，tag是标签的名字，attrs是标签的属性值字典
        self.CurrentDate=tag
        if tag=='country':
            name=attributes['name']
            print('-----------------------')
            print('country:',name)
    def characters(self,content):
        if self.CurrentDate=='rank':
            self.rank=content
        elif self.CurrentDate=='year':
            self.year = content
        elif self.CurrentDate=='neighbor':
            self.neighbor=content
    def endElement(self,tag):
        #endElement(name)方法,遇到XML结束标签时调用
        if self.CurrentDate=='rank':
            print('rank:',self.rank)
        elif self.CurrentDate=='year':
            print('year:',self.year)
        elif self.CurrentDate=='neighbor':
            print('neighbor:',self.neighbor)
        self.CurrentDate=''     #
if __name__=='__main__':
    parser=xml.sax.make_parser()     #创建一个xml读取器
    parser.setFeature(xml.sax.handler.feature_namespaces,0)
    handler=countryHandler()         #
    parser.setContentHandler(handler)
    parser.parse('countrys.xml')     #解析文档

Redis数据库入门

Redis数据库入门

NoSql(Not Only Sql)概念

  • NoSql是非关系型数据库的集合，如Redis、MongoDB、HBase…
  • NoSql产生背景，为解决三高而产生：数据库高并发大写；海量数据下的效率；高扩展和可用性
  • NoSQL优点：扩展方便，读写快速高效，数据类型灵活
  • NoSQL缺点：不支持sql语句

Redis简介

  • 由c语言编写，基于内存运行的并支持持久化，存储形式为key-value型
  • 单进程运行
    • redis服务程序是单进程运行，当多个客户端同时访问时，服务器的处理能力是会有一定的下降的。面对这种场景，我们可能会考虑在同一台服务器上开启多个Redis进程，Redis会对CPU存在竞争
  • Redis部署
    • 1.将redis安装包（此时最新版本是3.2.11）放入Linux服务器中
    • 2.解压redis，进入redis解压目录
    • 3.make
    • 4.make install，可指定安装目录，如make PREFIX="/opt/redis3" install
    • 5.之后安装目录中出现了bin目录，再将解压目录中的redis.conf放在与bin目录同级的目录下
    • 6.进入bin目录下，运行redis-server,出现层状方形小时运行成功
  • 在redis的bin目录下：
    • redis-server用来启动redis的工具，关闭该工具命令为shutdown
    • redis-benchmark检测redis在本机运行效率
    • redis-check-aof修复AOF持久化
    • redis-check-rdb修复RDB持久化文件
    • redis-cli命令行工具
  • Redis的多数据库（16个数据库：0-15 ）
    • Redis的多数据库在一定程度上是独立的,不支持自定义数据库名称，不能为每一个数据库设置独立密码
    • 切换数据库命令：select [dbindex]
    • 多数据库间移动数据：mv [key] [dbindex]
    • 清除数据库内数据：
      • 清空当前数据库数据：flushdb
      • 清空所有数据库数据：flushALL
    • 查看数据库内数据的数量：Dbsize
  • redis常用场景
    • 缓存
    • 排行榜类的应用
    • 实时分析系统

Redis参数配置

  • config命令：
    • 动态取出参数配置config get 参数名称，如config get port
    • 动态设置参数配置config set 参数名称
  • 配置分类：通用部分、快照、复制、安全、限制、追加、Lua脚本、慢日志、事件通知、高级配置、引入
  • Redis的基础配置（redis.conf）
    • notework:
      • bind: 指定要绑定的IP地址
      • protected-mode：询问是否开启保护模式；如不开启，bind会不起作用，密码也不起作用
      • port： 端口号，默认是6379，如果想开启多个Redis进程，需要指定不同的端口号
      • tcp-backlog：设置tcp的backlog的连接队列，即tcp的三次握手，未完成三次握手和已完成三次握手的队列总和
      • timeout：连接超时时间，默认是0
      • tcp-keepalive：检测keepalive的值，即健康检查，默认单位是秒。HA，默认值是0，表示不检测，建议设置为60秒
    • 通用配置：
      • daemonize：是否以后台进程的方式运行
      • pidfile: 指定pid存储的路径和文件，在正式使用时，一定会使用pidfile指定路径
    • 日志相关：
      • loglevel:四个级别
      • logfile：指定log日志文件位置,/dev/null
    • databases:
      • Redis启动后有多少个数据库
    • Redis限制部分配置[limits]
      • maxclients:客户端最大连接数，默认是10000
      • maxmemory：最大内存数，实际工作时，最大内存数一般都是50%左右的内存，一定不能超过70%
      • maxmemory-samples：样例数量
      • maxmemory-policy：设置内存的移除规则
        • volatile-lru：LRU算法进行移除，只针对设置了过期时间的值
        • allkeys-lru ：LRU算法
        • volatile-random：在过期集合中移除随机的key
        • allkeys-random：移除随机的key
        • volatile-ttl ：移除TTL最小的key，即数据的生命期
        • noeviction ：默认值，不进行移除

key操作

  • keys ：获得符合规则的键值列表，keys *
  • exists: 检测key是否存在，exists key名称
  • del: 单条删除key操作，del key名称
  • type:获取key对应的value值类型，type key名称
  • rename语句： rename oldKey newKey 不管新的key有没有都改名
  • renamenx: renamenx oldkey newkey 如果新的key存在，则不改名

数据类型

  • String类型
    • string是redis最基本的类型,可以包含任何数据。比如jpg图片或者序列化的对象
  • List类型
    • Redis 列表是简单的字符串列表，按照插入顺序排序。你可以添加一个元素到列表的头部（左边）或者尾部（右边）
  • Set类型
    • Redis的Set是string类型的无序集合。集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是O(1)
  • Hash类型
    • Redis hash 是一个键值(key=>value)对集合，是一个string类型的field和value的映射表，hash特别适合用于存储对象
  • Zset类型
    • Redis zset 和 set 一样也是string类型元素的集合,且不允许重复的成员。不同的是每个元素都会关联一个double类型的分数。redis正是通过分数来为集合中的成员进行从小到大的排序

分支管理

  • Git会串成一条时间线，这条时间线就是分支， 默认就是master
    HEAD并不是指向提交，而是指向master，指向最新版本

    v1  ==>  v2  ==> v3
                 |
                 master
                  |
                  HEAD
如果创建了新的分支，head将指向新的分支，如下
                        master
                          |
v1  ==>  v2  ==> v3  ==> v4
                          |
                        dev
                          |
                        HEAD

  • 基本命令

    • 查看分支：git branch（*表示你当前所处的branch是哪个）
    • 创建分支：git branch 分支名
    • 切换分支： git checkout 分支名
    • 创建并切换分支：git checkout -b 分支名
    • 合并分支：git merge 分支名 （需要切换到主分支或者依附分支再执行该命令）
    • 删除分支： git branch -d 分支名
    • 分支合并图：git log –graph –pretty=oneline –abbrev-commit

  • 解决冲突

    • 先在feature1 分支: a.txt中新增并提交‘AND simple’；
      后在master 分支: a.txt 中新增提交‘& simple’
      在master和分支上分别对a.txt做了不同修改之后，合并分支出现以下错误：

      git merge feature1
Auto-merging a.txt
CONFLICT (content): Merge conflict in a.txt
Automatic merge failed; fix conflicts and then commit the result.

    • 当Git无法自动合并分支时，就必须首先解决冲突，再提交，最后merge

  • 分支管理策略：

    • 1）master分支： 代码是稳定的，平时不能在上面干活的，仅仅是用来发布新版本用
    • 2）其他分支：是不稳定的，bug、feature可以创建新的分支进行开发，最后再合并到master即可

标签管理

  • 标签（tag），是版本库的一个快照；发布一个新版本时，通常先在版本库中打一个标签，无论在将来什么时候，取某个标签的版本即可

  • 基本命令

    git tag <tagname>创建标签
git tag查看所有标签
git tag -a <tagname> -m <desc>创建带有说明的标签
git show <tagname>查看标签的说明
git tag -d <tagname>删除标签

git push origin <tagname>推送一个本地标签
git push origin --tags推送全部未推送的本地标签

#如果tag已经推送到远程，要删除远程标签分2步：
1）删除本地tag:  git tag -d 本地标签名
2）git push origin :refs/tags/本地标签名

多人协作

  • 开源项目肯定都是很多人参与到社区中，通过协作完成
  • 多人协作常用操作
    • 克隆远程仓库：git clone xxxxxxx
    • 查看远程信息：git remove (-v)
    • 创建自己的分支：git branch xxxxxx
    • 推送分支：git push origin
    • 创建Pull Request

其他操作

    Git控制台颜色设置

git config --global color.status auto
git config --global color.branch auto

颜色设置关闭
git config --global color.status false

  • 忽略某些特殊文件不提交

    • 1）只需要在git项目的根目录下创建一个.gitignore
    • 2) 把需要提交的文件或者目录名称写到.gitignore
    • 3）如果你真正想提交的文件已经被包含在.gitignore里面了，那么就需要使用-f进行强制提交

re模块

re模块

常用匹配模式

  • 正则表达式是对字符串操作的一种逻辑公式，就是用事先定义好的一些特定字符及这些特定字符的组合，组成一个“规则字符串”，这个“规则字符串”用来表达对字符串的一种过滤逻辑

    \w     匹配字母数字及下划线
\W     匹配非字母数字及下划线
\s     匹配任意空白字符
\S     匹配任意非空白字符
\d     匹配任意数字，等价于[0-9]
\D     匹配任意非数字
^      匹配字符串开头
$      匹配字符串结尾
.      匹配除了换行符外的任意字符
*      匹配0个或者多个的表达式
+      匹配至少一个表达式
？     匹配最多一个表达式，非贪婪方式
{n}    精准匹配n个前面的表达式
{n,m}    匹配n到m次前面的表达式的片断
a|b    匹配a或者b
()     匹配括号内的表达式，也表示一个组

re.match(表达式,内容,匹配模式)

  • 尝试从字符串的起始位置匹配一个模式，如果不是起始位置匹配成功的话，返回none

  • 在Match对象上用group()方法提取子串

    import re
content='Hello 20180101 World_This is my festival year'
result=re.match(r'^Hello.*?(\d+).*year$',content)  #使用Python的r前缀，就不用考虑转义的问题了
print(result)
print(result.group(0))    #group(0) 永远是原始字符串
#想要获取content中的数字，如果不加问号，result为1，这是贪婪匹配；加了问号后，result为20180101，这是非贪婪匹配
print(result.group(1))    #result:20180101

  • 匹配模式

    import re
content='''Hello 20180101 World_This
        is my festival year'''
result=re.match(r'^Hello.*?(\d+).*year$',content,re.S)  #有换行符就用re.S
print(result.group(1))     #result:20180101

re.search()

  • 扫描整个字符串并返回第一个成功的匹配

    import re
content='Liuyongqian Hello 20180101 World_This is my festival year'
result=re.search(r'Hello.*year',content)
print(result)          #为匹配方便，一般使用search

re.findall()

  • 搜索字符串，以列表形式返回全部能匹配的子串

    import re
html='''
    <div>
        <ul>
            <li data-view='1'>
                <a href="/2.mp3" singer="齐秦">往事随风</a>
            </li>
            <li><a href="/3.mp3" singer="邓丽君">但愿人长久</a></li>
            <li><a href="/3.mp3" singer="水木年华">完美世界</a></li>
        </ul>
    </div>
'''
results=re.findall(r'<li.*?href=(.*?mp).*?singer=(.*?)>(.*?)</a>',html,re.S)
for result in results:
    print(result[0],result[1],result[2])

re.sub()

  • 替换字符串中每一个匹配的子串后返回替换后的字符串

    import re
result=re.sub(r'\d+','2018',content)
print(result)      #result:Hello 2018 World_This is my festival year

re.compile()

  • 将正则字符串编译成正则对象，以便于复用该匹配模式

    import re
content='''Hello 20180101 World_
        This is my festival year'''
pattern=re.compile(r'^He.*?(\d+).*?year$',re.S)
result=re.match(pattern,content)
print(result.group(1))       #result:20180101

Git入门

Git入门

简介

  • git是一款免费、开源的分布式版本控制系统，用于敏捷高效的处理任何或大或小的项目
    github是一个面向开源及私有软件项目的托管平台，因为只支持git作为唯一的版本库格式进行托管，故名github
  • Git和SVN的对比：
    • 1）Git是分布式的，SVN是集中式
    • 2）Git可离线完成大部分操作，SVN则必须与中央服务器进行网络交互
    • 3）Git有着优雅的分支和合并功能
    • 4）Git有着更强的撤销修改和修改版本历史的能力
    • 5）Git速度更快，效率更好
  • git的三个区
    • 工作区：编辑代码的地方
    • 暂存区：工作区与历史提交中间的缓存，代表的是要提交的工作状态
    • 历史仓库：commit指向的
  • Git管理的是修改，而不是文件

基本配置

  • 目前git for window的版本是2.16.2

    which -a git 查看git的安装位置
git version     查看git的版本

添加user.name和user.email
git config --global user.name fyuan925
git config --global user.email fyuan925@gmail.com

添加一个新的user.name
git config --global --add user.name kgc

得到属性的值
git config user.name
git config --get user.name

得到全局所有属性的值
git config --list --global

删除一个属性，并且是指定属性名进行删除
git config --global --unset user.name kgc

当只有一个属性值的时候，可以在删除时不指定值
git config --global --unset user.name

  • 关联github

    • 本地生成SSH KEYS

      先生成ssh，命令如下：
ssh-keygen -t rsa -C 'kgc@gmail.com'
进入ssh目录后将该目录下文件id_rsa.pub中的内容拷贝

    • 在github上设置SSH公钥

      进入github主页，点击右上角的setting，选择SSH and GPG keys;
新建一个ssh key，将公钥粘贴后点击Add SSH key
输入GitHub的密码进行验证，点击Confirm password

本地库连接远程库

  • 第一种方式：先有本地仓库，后有远程仓库

    在本地仓库中执行以下命令，同步信息
git remote add origin git@github.com:fangyuan925/kgc_git.git
git push -u origin master
第一次使用-u，后面不再使用

  • 第二种方式：先有远程仓库，再clone到本地（最常见的，一般都是先在GitHub上创建仓库，可以选择初始化选项，然后开发人员再clone到本地）

    git clone xxxxxx
git push -u origin master
第一次使用-u，后面不再使用

基本命令

  • 在本地新建仓库

    git init      新建文件夹后用此命令初始化仓库
vi readme.txt 添加一个readme.txt
git status  查看当前版本库的状态
git add filename    使用git add命令告诉git，把该文件添加到仓库
git commit -m 'comment'  使用git commit命令告诉git，把文件提交到仓库,可设置注释内容

git add     可以反复添加多个
git diff filename     查看修改的内容
git log      可以查看所有的提交历史记录
git log filename     仅查看指定文件名的提交历史记录
git log --pretty=oneline filename
d085ee8434392d2cf88b48592f29fbeefb0818f7  此为commit id（版本号） SHA-1 Hash计算出来一个16进制的值

  • 版本回退

    前提：知道当前版本是哪个， HEAD表示当前版本
回退到上一个版本： HEAD^   一个^就表示前一个版本， 如果N个前版本可以用HEAD~N

回退命令：git reset --hard HEAD^
回退之后你后悔了又想回到后退之前的那个版本： git reset --hard xxxxxxxxx
xxxxxxxxx  前面可以通过 git log filename

3commit   <== HEAD
2commit
1commit

执行了git reset --hard HEAD^之后
3commit   
2commit  <== HEAD
1commit

git reset：版本穿梭， 可以使用git log查看提交历史，以便确定回退/穿梭到哪个版本     
git reflog：查看命令历史，以便确定要回到未来/过去的哪个版本

  • 撤销修改

    没有提交到暂存区时，其实就在工作区： git checkout – filename
已经提交到暂存区了： 分两步：1）git reset HEAD file 2）git checkout -- filename

  • 删除文件

    手误删除，如何恢复： git checkout -- filename
真删除版本库的文件： 
    1) git rm filename
    2) git commit -m 'xxx'

Python之OS操作

Python之OS操作

简介

  • os 模块在运维工作中是很常用的一个模块。通过os模块调用系统命令
  • os模块可以跨平台使用
  • 在 import os的时候，建议使用import os而非from os import *。这样可以避免os.open()不会覆盖内置函数open()

模块详解

  • os.name()判断系统类型

    import os
print(os.name)      #linux和Unix系统会返回`posix`;windows 系统会返回`nt`

  • os.system()执行系统命令的模块，返回命令执行的状态码，开启一个子shell执行命令

    cmd='ipconfig'
retval=os.system(cmd)
print(retval)          #result: 0,表示命令执行成功，否则为失败

  • os.listdir(path)打印指定目录的文件,返回一个列表

    a=os.listdir('E:\eclipse')
print(a)

  • os.chdir(new_path)改变当前路径

    print(os.getcwd())          #result:E:\selenium prt\0311
os.chdir('E:\\selenium prt\\0312')
print(os.getcwd())          #result:E:\selenium prt\0312

  • os.getcwd()返回当前绝对路径,返回类型为str

    a=os.getcwd()
print(a)

  • os.mkdir(path[,mode=0755])默认权限是0755，创建目录

    os.mkdir(r'E:\selenium prt\0311\0002')
print(os.listdir(r'E:\selenium prt\0311'))

  • os.rmdir(path)删除目录，只能删除目录

    os.rmdir(r'E:\selenium prt\0311\0002')
print(os.listdir(r'E:\selenium prt\0311'))

  • 创建名为text的txt文件

    def txt(name,text):
    a=name+'.txt'
    file=open(a,'w')
    file.write(text)
    file.close()
txt('test','Hello，2018！')

  • os.remove(path)删除文件，只能删除文件

    os.remove(r'E:\selenium prt\0311\test.txt')
print(os.listdir(r'E:\selenium prt\0311'))

  • os.path该模块主要针对路径的操作

    • os.path.abspath(‘.’)返回当前绝对路径
    • os.path.basename(r’E:\selenium prt\0311’)返回当前文件夹名
    • os.path.dirname(r’E:\selenium prt\0311’)返回文件路径，不包含当前文件夹名
    • os.path.exists(r’E:\selenium prt\0311’)判断路径是否存在，存在返回True，不存在返回False
    • os.path.join(path,name):连接目录与文件名或目录

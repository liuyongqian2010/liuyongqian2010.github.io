mysql5.6压缩版的安装配置

mysql5.6压缩版的安装配置

下载

  • 下载地址：http://ftp.ntu.edu.tw/MySQL/Downloads/MySQL-5.6/
  • 当前下载版本：mysql-5.6.41-winx64.zip

解压

  • 解压之后可以将该文件夹改名（比如：mysql5.6），放到合适的位置（比如：D:\mysql5.6）

配置

  • 环境变量，在path中增加变量D:\mysql5.6\bin

  • 配置文件，mysql-5.6.1X默认的配置文件是在D:\mysql5.6\my-default.ini中，或者自己建立一个my.ini文件，在其中修改或者添加配置信息：

    [mysqld] 
basedir=D:\mysql5.6（mysql所在目录） 
datadir=D:\mysql5.6\data （mysql所在目录\data）

安装

  • 以管理员身份运行cmd，进入mysql的bin目录下（D:\mysql5.6\bin）
  • 接着输入mysqld -install并回车进行数据库安装

启动

  • 以管理员身份运行cmd，输入net start mysql并回车，服务启动成功
  • 接着输入mysql -u root -p命令可以进入mysql操作界面

设置密码

  • 第一次配置并启动mysql服务时，默认账号是root，密码为空；如果需要设置密码，可以通过下面两种方式进行设置

    • 第一种：用root 进入mysql后

      mysql>set password =password('你的密码');
mysql>flush privileges;

    • 第二种：进入mysql库修改user表

      mysql>use mysql;
mysql>update user set password=password('你的密码') where user='root'; 
mysql>flush privileges;

JDK的安装部署

JDK的安装部署

JDK下载

  • JDK是ORACLE提供免费下载使用的，官网地址：https://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html
  • 一般选择Java SE版本即可，企业版的选择Java EE。当前下载的版本是jdk-8u181-windows-x64

JDK安装

  • 安装过程中会出现两次安装提示 ：第一次是安装 jdk ，第二次是安装 jre 。建议两个都安装在同一个java文件夹中的不同文件夹中（java文件夹下新建jdk、jre两个文件夹）

JDK配置

  • 新建系统变量JAVA_HOME（指向jdk目录，之所以这样命名是因为eclipse、tomcat等均通过java_home变量名寻找jdk目录）

    变量名：JAVA_HOME
变量值：C:\Program Files\Java\jdk1.8.0_181

  • 新建系统变量CLASSPATH（类搜索路径，import导入的类均在此路径内;变量值前面有点号和分号，后边结尾也有分号）

    变量名：CLASSPATH
变量值：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar；

  • 添加环境变量到path中（命令搜索路径，javac.exe、java.exe等命令均在此路径内）

    变量值：%JAVA_HOME%\bin；

JDK测试

  • Win+R——>输入cmd——>输入java -version或者javac或者java——>回车Enter

重要

  • 我按照上述步骤配置后，输入“javac”提示不是内部命令

  • 后来我直接只在“path”中添加三个变量即可

    C:\Program Files\Java\jdk1.8.0_181\bin
C:\Program Files\Java\jdk1.8.0_181\lib\dt.jar
C:\Program Files\Java\jdk1.8.0_181\lib\tools.jar

nmon工具的使用

nmon工具的使用

Nmon简介

  • Nmon (Nigel’s Monitor)是由IBM 提供、免费监控 AIX 系统与 Linux 系统资源的工具。该工具可将服务器系统资源耗用情况收集起来并输出一个特定的文件,并可利用 excel 分析工具（nmon analyser）进行数据的统计分析
  • nmon 工具可以为 AIX 和 Linux 性能专家提供监视和分析性能数据的功能，其中包括：
    • CPU 使用率、内存使用情况、内核统计信息和运行队列信息、磁盘 I/O 速度、传输和读/写比率
    • 文件系统中的可用空间、磁盘适配器、网络 I/O 速度及传输和读/写比率
    • 页面空间和页面速度、CPU 和 AIX 规范、消耗资源最多的进程、IBM HTTP Web 缓存
    • 计算机详细信息和资源、网络文件系统 (NFS)

nmon安装使用

  • nmon下载：http://nmon.sourceforge.net/pmwiki.php?n=Site.Download

  • nmon安装：我的系统是centos7，下载的版本是nmon16e_mpginc.tar.gz。解压后，根据不同平台初始化对应平台的nmon工具，初始化命令如下

    chmod u+x nmon_x86_64_centos7
cp nmon_x86_64_centos7 /usr/local/bin/nmon

  • nmon使用：在任意目录下运行’nmon’即可，采集数据的命令如下

    nmon -s10 -c60 -f -m /home/
参数解释：
-s10          每 10 秒采集一次数据
-c60          采集 60 次，即为采集十分钟的数据
-f            生成的数据文件名中包含文件创建的时间。
-m            生成的数据文件的存放目

nmon analyser安装使用

  • nmon analyser下载：https://www.ibm.com/developerworks/community/wikis/home?lang=en#!/wiki/Power%20Systems/page/nmon_analyser
  • nmon analyser使用：
    • 当前所下载的版本是nmon analyser v55.zip，解压后放在windows系统任意目录下即可
    • 将之前生成的 nmon 数据文件传到 Windows 机器上，用 Excel 打开分析工具nmon analyser v55.xlsm 。点击 Excel 文件中的 “Analyze nmon data” 按钮，选择 nmon 数据文件，这样就会生成一个分析后的结果文件

  • 本文参照博客Linux下安装使用NMON监控、分析系统性能
  • 关于nmon结果分析可以参照这两篇不错的博客
    • 一篇文章学会使用nmon

Redis安装与部署

Redis安装与部署

一.redis介绍

  • 1.Redis是当前比较热门的NOSQL系统之一，它是一个key-value存储系统
  • 2.Redis数据都是缓存在计算机内存中，会周期性的把更新的数据写入磁盘或者把修改操作写入追加的记录文件，实现数据的持久化

二.redis安装

  • 1.目前的版本是redis-4.0.10.tar.gz

  • 2.通过远程工具将该压缩包拷贝到Linux服务器中，并执行解压操作

    将压缩包解压到/usr/local下执行：
tar -zvxf redis-4.0.10.tar.gz -C /usr/local/
重命名解压后的文件夹名称执行：
mv redis-4.0.10 redis

  • 3.对解压后的文件进行编译、安装

    cd /usr/local/redis下执行：make
cd src下执行：make install

  • 4.安装完成，界面如下所示

    [root@bogon src]# make install
    CC Makefile.dep

Hint: It's a good idea to run 'make test' ;)

    INSTALL install
    INSTALL install
    INSTALL install
    INSTALL install
    INSTALL install

三.redis部署

  • 1.为方便管理，将Redis文件中的conf配置文件和常用命令移动到统一文件中

    创建bin和etc文件夹
cd /usr/local/redis下后执行: mkdir bin    mkdir etc
执行文件移动命令
在redis目录下执行：mv redis.conf etc/
进入src目录下执行：cd src/
接着执行：mv mkreleasehdr.sh redis-benchmark redis-check-aof redis-cli redis-server /usr/local/redis/bin

  • 2.后台启动redis服务

    • 编辑redis.conf文件，将daemonize属性改为yes（表明需要在后台运行）

      cd /usr/local/redis/etc
vi redis.conf  在GENERAL下修改daemonize属性为yes

    • 使用配置文件redis.conf来启动redis服务

      cd /usr/local/redis/bin
执行./redis-server /usr/local/redis/etc/redis.conf

    • 服务端启动完成后执行redis-cli启动客户端，查看端口-6379

  • 3.关闭redis服务器

    查看redis进程ps -ef|grep redis
sudo kill 进程

    cd /mnt/project/apache-tomcat-7.0.79-info

    在XFTP5中，远程覆盖替换jeeplus.properties，路径是：
/mnt/project/apache-tomcat-7.0.79-info/webapps/ROOT/WEB-INF/classes

开服务

  • 启动服务

    cd /mnt/project/apache-tomcat-7.0.79-info
sudo ./bin/startup.sh

看日志

  • 查看服务器输出内容

    tail -f /mnt/project/apache-tomcat-7.0.79-info/logs/catalina.out

Linux上部署java项目

Linux上部署java项目

关闭防火墙

  • 目前安装的linux版本是CentOS7
  • 永久关闭防护墙 systemctl disable firewalld

安装JDK

  • 目前版本是jdk-7u79-linux-x64.tar.gz

  • 安装步骤：

    • 1.在/usr目录下mkdir java，把jdk安装包放到该文件夹下并解压tar -zvxf jdk-7u79-linux-x64.tar.gz

    • 2.修改系统环境变量vi /etc/profile在文件尾部添加如下内容：

      JAVA_HOME=/usr/java/jdk1.7.0_79
PATH=$PATH:$JAVA_HOME/bin
CLASSPATH=.:$JAVA_HOME/lib
export JAVA_HOME PATH CLASSPATH

      3.之后重新加载，使其生效，命令为：source /etc/profile
      4.查看是否安装成功，命令为：java -version

安装Tomcat

  • 目前的版本是apache-tomcat-7.0.79.tar.gz

  • 安装步骤：

    • 1.在/usr目录下mkdir tomcat，把Tomcat安装包放到该文件夹下并解压tar -zvxf apache-tomcat-7.0.79.tar.gz

    • 2.修改系统环境变量vi /etc/profile，如下：

      JAVA_HOME=/usr/java/jdk1.7.0_79
CATALINA_HOME=/usr/tomcat/apache-tomcat-7.0.79
PATH=$PATH:$JAVA_HOME/bin:$CATALINA_HOME/bin
CLASSPATH=.:$JAVA_HOME/lib:$CATALINA_HOME/lib
export JAVA_HOME CATALINA_HOME PATH CLASSPATH
      • 3.之后重新加载，使其生效，命令为：source /etc/profile

      • 4.启动Tomcat，并查看日志

        cd /usr/tomcat/apache-tomcat-7.0.79 
执行 ./bin/startup.sh命令即可启动

      • 5.关闭tomcat

        ps -ef |grep tomcat  
查看进程后杀掉进程sudo kill ***

安装MySQL

  • 目前的版本是mysql-5.6.39-linux-glibc2.12-x86_64.tar.gz

  • 安装步骤：

    • 1.将安装包拷贝到/usr/local目录下并解压tar -zvxf mysql-5.6.39-linux-glibc2.12-x86_64.tar.gz之后mv mysql-5.6.39-linux-glibc2.12-x86_64 mysql

    • 2.均在/usr/local下执行：

      添加mysql用户组，执行 groupadd mysql
为mysql用户组添加mysql用户，执行useradd -r -g mysql mysql -d /usr/local/mysql
为mysql用户设置密码，执行passwd mysql    密码可以是mysql
将文件及其子文件的所有者设为mysql组的mysql用户，执行chown -R mysql:mysql /usr/local/mysql

    • 3.切换到mysql用户即su mysql后安装数据库

      /usr/local/mysql/scripts/mysql_install_db --user=mysql --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data

    • 4.安装mysql数据库结束后exit回到/usr/local目录下

    • 5.cd mysql/support-files后复制mysql配置文件到系统配置目录下,执行cp my-default.cnf /etc/my.cnf

    • 6.在mysql/support-files下将mysql添加为系统服务依次执行

      cp mysql.server /etc/init.d/mysql                                                                          chkconfig --list|grep mysql                                                                           chkconfig --add mysql
查看mysql服务是否添加成功chkconfig --list|grep mysql

    • 7.添加环境变量，vi /etc/profile后增加

      JAVA_HOME=/usr/java/jdk1.7.0_79
CATALINA_HOME=/usr/tomcat/apache-tomcat-7.0.79
MYSQL_HOME=/usr/local/mysql
PATH=$PATH:$JAVA_HOME/bin:$CATALINA_HOME/bin:$MYSQL_HOME/bin
CLASSPATH=.:$JAVA_HOME/lib:$CATALINA_HOME/lib
export JAVA_HOME CATALINA_HOME MYSQL_HOME PATH CLASSPATH

    • 8.之后重新加载，使其生效，执行source /etc/profile

    • 9.修改mysql配置文件vi /etc/my.cnf，修改内容如下

      #These are commonly set,remove the # and set as required
basedir=/usr/local/mysql
datadir=/usr/local/mysql/data
port=3306
server_id=1
socket=/tmp/mysql.sock

    • 10.保存退出后，启动mysql服务

      启动mysql服务，执行service mysql start
查看mysql服务，执行service mysql status

    • 11.其他必须操作

      root账户默认是没有密码的，如果需要修改root账户密码：
第一种方式（不建议采用，会将linux系统密码也同步修改）：
执行mysqladmin -u root password 'admin'
第二种方式（登录mysql系统，执行mysql -u root -p）：
Enter password: 【输入原来的密码，首次为空】
mysql>use mysql;
mysql> update user set password='新的密码' where user='root';
mysql> flush privileges;
授权允许所有外部链接访问执行 GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY 'admin' WITH
GRANT OPTION; （注：admin为登录用户的密码）
刷新权限执行：FLUSH PRIVILEGES;

  • 2.在该目录下，修改conf/server.xml，具体如下：

    在<Host>下添加<Context path="" docBase="../jeeplus.war"/>

  • 3.全量部署时需要将webapps下的ROOT目录删除，第一次部署时要把ROOT目录删除先再重启服务

    # -----------------------------------------------------------------------------
JAVA_OPTS="-server -XX:PermSize=128M -XX:MaxPermSize=256M -XX:+HeapDumpOnOutOfMemoryError -XX:+PrintGCDetails -XX:+PrintGCDateStamps -XX:+PrintHeapAtGC -Xloggc:gc.log"
# OS specific support.  $var _must_ be set to either true or false.

  • 5.在该目录下，修改webapps/ROOT/webpage/include/taglib.jsp,就是关于js文件获取方式，具体注销内容如下：

    <c:set var="ctxStatic" value="${pageContext.request.contextPath}/static"/>

<c:set var="ctxp" value="${pageContext.request.contextPath}" />

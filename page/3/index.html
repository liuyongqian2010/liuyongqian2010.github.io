nmon工具的使用

nmon工具的使用

Nmon简介

  • Nmon (Nigel’s Monitor)是由IBM 提供、免费监控 AIX 系统与 Linux 系统资源的工具。该工具可将服务器系统资源耗用情况收集起来并输出一个特定的文件,并可利用 excel 分析工具（nmon analyser）进行数据的统计分析
  • nmon 工具可以为 AIX 和 Linux 性能专家提供监视和分析性能数据的功能，其中包括：
    • CPU 使用率、内存使用情况、内核统计信息和运行队列信息、磁盘 I/O 速度、传输和读/写比率
    • 文件系统中的可用空间、磁盘适配器、网络 I/O 速度及传输和读/写比率
    • 页面空间和页面速度、CPU 和 AIX 规范、消耗资源最多的进程、IBM HTTP Web 缓存
    • 计算机详细信息和资源、网络文件系统 (NFS)

nmon安装使用

  • nmon下载：http://nmon.sourceforge.net/pmwiki.php?n=Site.Download

  • nmon安装：我的系统是centos7，下载的版本是nmon16e_mpginc.tar.gz。解压后，根据不同平台初始化对应平台的nmon工具，初始化命令如下

    chmod u+x nmon_x86_64_centos7
cp nmon_x86_64_centos7 /usr/local/bin/nmon

  • nmon使用：在任意目录下运行’nmon’即可，采集数据的命令如下

    nmon -s10 -c60 -f -m /home/
参数解释：
-s10          每 10 秒采集一次数据
-c60          采集 60 次，即为采集十分钟的数据
-f            生成的数据文件名中包含文件创建的时间。
-m            生成的数据文件的存放目

nmon analyser安装使用

  • nmon analyser下载：https://www.ibm.com/developerworks/community/wikis/home?lang=en#!/wiki/Power%20Systems/page/nmon_analyser
  • nmon analyser使用：
    • 当前所下载的版本是nmon analyser v55.zip，解压后放在windows系统任意目录下即可
    • 将之前生成的 nmon 数据文件传到 Windows 机器上，用 Excel 打开分析工具nmon analyser v55.xlsm 。点击 Excel 文件中的 “Analyze nmon data” 按钮，选择 nmon 数据文件，这样就会生成一个分析后的结果文件

Redis安装与部署

Redis安装与部署

一.redis介绍

  • 1.Redis是当前比较热门的NOSQL系统之一，它是一个key-value存储系统
  • 2.Redis数据都是缓存在计算机内存中，会周期性的把更新的数据写入磁盘或者把修改操作写入追加的记录文件，实现数据的持久化

二.redis安装

  • 1.目前的版本是redis-4.0.10.tar.gz

  • 2.通过远程工具将该压缩包拷贝到Linux服务器中，并执行解压操作

    将压缩包解压到/usr/local下执行：
tar -zvxf redis-4.0.10.tar.gz -C /usr/local/
重命名解压后的文件夹名称执行：
mv redis-4.0.10 redis

  • 3.对解压后的文件进行编译、安装

    cd /usr/local/redis下执行：make
cd src下执行：make install

  • 4.安装完成，界面如下所示

    [root@bogon src]# make install
    CC Makefile.dep

Hint: It's a good idea to run 'make test' ;)

    INSTALL install
    INSTALL install
    INSTALL install
    INSTALL install
    INSTALL install

三.redis部署

  • 1.为方便管理，将Redis文件中的conf配置文件和常用命令移动到统一文件中

    创建bin和etc文件夹
cd /usr/local/redis下后执行: mkdir bin    mkdir etc
执行文件移动命令
在redis目录下执行：mv redis.conf etc/
进入src目录下执行：cd src/
接着执行：mv mkreleasehdr.sh redis-benchmark redis-check-aof redis-cli redis-server /usr/local/redis/bin

  • 2.后台启动redis服务

    • 编辑redis.conf文件，将daemonize属性改为yes（表明需要在后台运行）

      cd /usr/local/redis/etc
vi redis.conf  在GENERAL下修改daemonize属性为yes

    • 使用配置文件redis.conf来启动redis服务

      cd /usr/local/redis/bin
执行./redis-server /usr/local/redis/etc/redis.conf

    • 服务端启动完成后执行redis-cli启动客户端，查看端口-6379

  • 3.关闭redis服务器

    查看redis进程ps -ef|grep redis
sudo kill 进程

JAVA项目更新部署

JAVA项目更新部署

先备份

  • 将当前的war包备份

    cd /mnt/project/apache-tomcat-7.0.79-info 
sudo cp -f ./jeeplus.war /mnt/project/copyInfo/

断服务

  • 查看进程后杀掉该Tomcat进程

    cd /mnt/project/apache-tomcat-7.0.79-info 
ps -ef |grep tomcat  
sudo kill ***

打包war

  • 使用软件有两个，一个是sourcetree，负责从git上拉取代码；另一个是java开发编辑工具ideaIU，负责调试、打包

  • 在本地需要有一个项目工程文件的存放位置，比如C:\ruihuaoa，在先期，需要将工程代码全部拉取到本地，命令如下：

    cd c/ruihuaoa
git clone http://code.gangwaninfo.com/ruihuaoa/jeeplus_maven.git

  • 在日常的更新部署中，先删除本地的war包

    在本地C:\ruihuaoa\jeeplus_maven\target下删除war文件

  • 之后使用sourcetree拉取代码到本地后，如果拉取的代码中有关于js的代码，可以先在ideaIU中运行一次，之后再打包

    Build--Build Artifact--jeeplus:war--build

删除替换

  • 1.上传新的war包

    cd /mnt/project/apache-tomcat-7.0.79-info
sudo rm -rf jeeplus.war      
sudo rm -rf ./webapps/ROOT
之后使用XFTP5远程上传新的war包到/mnt/project/apache-tomcat-7.0.79-info下

  • 2.启动一次服务，在webapps下生成新的ROOT文件夹后，再断了服务

    cd /mnt/project/apache-tomcat-7.0.79-info
sudo ./bin/startup.sh

  • 3.要以普通人权限替换配置文件，需要赋予操作权限

    cd /mnt/project/apache-tomcat-7.0.79-info
sudo chmod -R 777 ../apache-tomcat-7.0.79-info

  • 4.替换配置文件jeeplus.properties

    在XFTP5中，远程覆盖替换jeeplus.properties，路径是：
/mnt/project/apache-tomcat-7.0.79-info/webapps/ROOT/WEB-INF/classes

开服务

  • 启动服务

    cd /mnt/project/apache-tomcat-7.0.79-info
sudo ./bin/startup.sh

看日志

  • 查看服务器输出内容

    tail -f /mnt/project/apache-tomcat-7.0.79-info/logs/catalina.out

Linux上部署java项目

Linux上部署java项目

关闭防火墙

  • 目前安装的linux版本是CentOS7
  • 永久关闭防护墙 systemctl disable firewalld

安装JDK

  • 目前版本是jdk-7u79-linux-x64.tar.gz

  • 安装步骤：

    • 1.在/usr目录下mkdir java，把jdk安装包放到该文件夹下并解压tar -zvxf jdk-7u79-linux-x64.tar.gz

    • 2.修改系统环境变量vi /etc/profile在文件尾部添加如下内容：

      JAVA_HOME=/usr/java/jdk1.7.0_79
PATH=$PATH:$JAVA_HOME/bin
CLASSPATH=.:$JAVA_HOME/lib
export JAVA_HOME PATH CLASSPATH

      3.之后重新加载，使其生效，命令为：source /etc/profile
      4.查看是否安装成功，命令为：java -version

安装Tomcat

  • 目前的版本是apache-tomcat-7.0.79.tar.gz

  • 安装步骤：

    • 1.在/usr目录下mkdir tomcat，把Tomcat安装包放到该文件夹下并解压tar -zvxf apache-tomcat-7.0.79.tar.gz

    • 2.修改系统环境变量vi /etc/profile，如下：

      JAVA_HOME=/usr/java/jdk1.7.0_79
CATALINA_HOME=/usr/tomcat/apache-tomcat-7.0.79
PATH=$PATH:$JAVA_HOME/bin:$CATALINA_HOME/bin
CLASSPATH=.:$JAVA_HOME/lib:$CATALINA_HOME/lib
export JAVA_HOME CATALINA_HOME PATH CLASSPATH
      • 3.之后重新加载，使其生效，命令为：source /etc/profile

      • 4.启动Tomcat，并查看日志

        cd /usr/tomcat/apache-tomcat-7.0.79 
执行 ./bin/startup.sh命令即可启动

      • 5.关闭tomcat

        ps -ef |grep tomcat  
查看进程后杀掉进程sudo kill ***

安装MySQL

  • 目前的版本是mysql-5.6.39-linux-glibc2.12-x86_64.tar.gz

  • 安装步骤：

    • 1.将安装包拷贝到/usr/local目录下并解压tar -zvxf mysql-5.6.39-linux-glibc2.12-x86_64.tar.gz之后mv mysql-5.6.39-linux-glibc2.12-x86_64 mysql

    • 2.均在/usr/local下执行：

      添加mysql用户组，执行 groupadd mysql
为mysql用户组添加mysql用户，执行useradd -r -g mysql mysql -d /usr/local/mysql
为mysql用户设置密码，执行passwd mysql    密码可以是mysql
将文件及其子文件的所有者设为mysql组的mysql用户，执行chown -R mysql:mysql /usr/local/mysql

    • 3.切换到mysql用户即su mysql后安装数据库

      /usr/local/mysql/scripts/mysql_install_db --user=mysql --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data

    • 4.安装mysql数据库结束后exit回到/usr/local目录下

    • 5.cd mysql/support-files后复制mysql配置文件到系统配置目录下,执行cp my-default.cnf /etc/my.cnf

    • 6.在mysql/support-files下将mysql添加为系统服务依次执行

      cp mysql.server /etc/init.d/mysql                                                                          chkconfig --list|grep mysql                                                                           chkconfig --add mysql
查看mysql服务是否添加成功chkconfig --list|grep mysql

    • 7.添加环境变量，vi /etc/profile后增加

      JAVA_HOME=/usr/java/jdk1.7.0_79
CATALINA_HOME=/usr/tomcat/apache-tomcat-7.0.79
MYSQL_HOME=/usr/local/mysql
PATH=$PATH:$JAVA_HOME/bin:$CATALINA_HOME/bin:$MYSQL_HOME/bin
CLASSPATH=.:$JAVA_HOME/lib:$CATALINA_HOME/lib
export JAVA_HOME CATALINA_HOME MYSQL_HOME PATH CLASSPATH

    • 8.之后重新加载，使其生效，执行source /etc/profile

    • 9.修改mysql配置文件vi /etc/my.cnf，修改内容如下

      #These are commonly set,remove the # and set as required
basedir=/usr/local/mysql
datadir=/usr/local/mysql/data
port=3306
server_id=1
socket=/tmp/mysql.sock

    • 10.保存退出后，启动mysql服务

      启动mysql服务，执行service mysql start
查看mysql服务，执行service mysql status

    • 11.其他必须操作

      root账户默认是没有密码的，如果需要修改root账户密码：
第一种方式（不建议采用，会将linux系统密码也同步修改）：
执行mysqladmin -u root password 'admin'
第二种方式（登录mysql系统，执行mysql -u root -p）：
Enter password: 【输入原来的密码，首次为空】
mysql>use mysql;
mysql> update user set password='新的密码' where user='root';
mysql> flush privileges;
授权允许所有外部链接访问执行 GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY 'admin' WITH
GRANT OPTION; （注：admin为登录用户的密码）
刷新权限执行：FLUSH PRIVILEGES;

安装redis

  • 目前版本是redis-4.0.10.tar.gz
  • 安装步骤：
    • 1.cd /usr/local/下，解压该文件并进入redis-4.0.10下的redis目录
    • 2.编译make，之后会有对应的src、conf等文件夹
    • 3.进入src文件夹执行make install进行安装
    • 4.接下来是部署redis（redis的安装部署，会在之后不久的博客中单独述说）

war包部署与配置

  • 1.将jeeplus.war包拷到tomcat根目录下，即/usr/tomcat/apache-tomcat-7.0.79下

  • 2.在该目录下，修改conf/server.xml，具体如下：

    在<Host>下添加<Context path="" docBase="../jeeplus.war"/>

  • 3.全量部署时需要将webapps下的ROOT目录删除，第一次部署时要把ROOT目录删除先再重启服务

  • 4.在该目录下，修改bin\catalina.sh，就是增加一句话JAVA_OPTS：

    # -----------------------------------------------------------------------------
JAVA_OPTS="-server -XX:PermSize=128M -XX:MaxPermSize=256M -XX:+HeapDumpOnOutOfMemoryError -XX:+PrintGCDetails -XX:+PrintGCDateStamps -XX:+PrintHeapAtGC -Xloggc:gc.log"
# OS specific support.  $var _must_ be set to either true or false.

  • 5.在该目录下，修改webapps/ROOT/webpage/include/taglib.jsp,就是关于js文件获取方式，具体注销内容如下：

    <c:set var="ctxStatic" value="${pageContext.request.contextPath}/static"/>
<%--<c:set var="ctxStatic" value="${fns:getOSSUrl()}"/> --%>
<c:set var="ctxp" value="${pageContext.request.contextPath}" />

  • 6.在该目录下，修改webapps/ROOT/WEB-INF/classes/jeeplus.properties,关于数据库、redis缓存的内容：

    #mysql database setting
jdbc.type=mysql
jdbc.driver=com.mysql.jdbc.Driver
jdbc.url=jdbc:mysql://127.0.0.1:3306/ruihuaoa-activity-info?useUnicode=true&characterEncoding=utf-8&allowMultiQueries=true
jdbc.username=root
jdbc.password=gangwan123

#redis settings
redis.keyPrefix=jeeplus
redis.host=127.0.0.1
redis.port=6379

综述

  • 进行整个库的导出导入时使用mongodump和mongorestore，这一对组合操作的数据是BSON格式，进行大量dump和restore时效率较高
  • 进行单个集合导出导入时使用mongoexport和mongoimport，这一对组合操作的数据是JSON格式，可读性较高

备份mongodump

  • mongodump是一个用于导出二进制数据库内容的实用工具，它导出的bson文档中只会包含着集合文档等信息，不包括索引信息（索引信息会单独导出），所以还原后，索引必须重建（这个不用担心，使用mongorestore会自动重建mongodump生成的索引信息）

  • 常用参数

    -h localhost:27017  #指定mongodb要连接的主机名及端口号，默认值为：localhost:27017 
-d <database>    #指定要备份的数据库，如果不指定，默认所有数据库
-c <collection>  #指定要备份的集合
-o <path>        #指定导出数据的目录路径
------------------------------------------
示例：将mytest数据库中的user集合导出到E:\test目录下
>>>mongodump -d mytest -c user -o E:\test

还原mongorestore

  • mongorestore用来导入数据到MongoDB实例中，3.0.0版本以上支持通过标准输入流来导入数据

  • 常用参数

    -d <database> #指定要还原的数据库
-c <collection>     #指定要还原的集合
--drop     #还原集合前会先从目标数据库中删除集合，不会删除不在备份中的集合
<path>    #要还原的数据文件路径，该参数必须是mongodbrestore命令的最后一个参数
------------------------------------------------
示例：通过user.bson文件还原mytest数据库中的user集合，并在还原之前进行删除
>>>mongorestore --drop -d mytest -c user E:\test\user.bson

导出mongoexport

  • 常用参数

    -d <database>      #指定要在哪个数据库上运行命令
-c <collection>    #指定要导出的集合
-f <field1,field2...>  #指定导出时只导出一个或者多个字段，导出多个时，需要用逗号分隔；当字段中有空格时，需要用英文引号括起来
-o <path-file>      #指定要导出的文件路径（含文件名）
-------------------------------------------------
示例：导出mytest数据库中的user集合到user.json文件中
>>>mongoexport -d mytest -c user -o E:\test\user.json

导入mongoimport

  • 常用参数

    --type <json|csv|tsv>  #要导入的文件类型，另外支持tsv
--headerline           #使用第一行作为字段名称
--mode <insert|upsert|merge>   #指定导入过程中，如何应对数据库文档与导入文件中的文档匹配的情况，默认会使用_id字段对比；<插入|替换数据库中的文档|合并>
------------------------------------------
示例：从user.json文件导入到mytest数据库中的user集合，并在之前进行删除
>>>mongoimport --drop -d mytest -c user --file E:\test\user.json

大数据的一些函数-02

  • parallelize

    from pyspark import SparkContext,SparkConf

conf=SparkConf().setAppName('demoPrj').setMaster('local[*]')
sc = SparkContext.getOrCreate(conf)
#用 python list 初始化
data =[1,2,3,4,5,6,7,8,9]
rdd = sc.parallelize(data,numSlices=4)       #parallelize并行化数据，转化为RDD;numSlices为分块数目，根据集群数进行分块
print(rdd.collect())             #collect返回RDD所有元素
print(rdd.getNumPartitions())    #返回分块数
print(rdd.glom().collect())      #glom，说明rdd中的元素已经变成了分片映射的列表
print(rdd.first())
sc.stop()
---------------------------------

4
[[1, 2], [3, 4], [5, 6], [7, 8, 9]]
1

  • 使用多个文本文件，进行词频统计

    sc = SparkContext.getOrCreate(conf)
#多个文本文件获取到一个RDD里面
filesRDD = sc.wholeTextFiles('file:///home/hadoop/*.txt')
#文件内容RDD
fileConRDD = filesRDD.map(lambda x:x[1])

    return x.split('\n')
#对每个文件内容做映射,结果是多个文件内容列表

strRDD = fileConRDD.map(sp)

wordDictRDD = wordRDD.map(lambda x:(x,1))

r = wordDictRDD.reduceByKey(lambda x,y:x+y)
print(strRDD.collect())
print(wordRDD.collect())
print(wordDictRDD.collect())
print(r.collect())
sc.stop()

Centos7安装MySQL5.6

Centos7安装MySQL5.6

安装MySQL

  • 1.下载安装包mysql-5.6.40-linux-glibc2.12-x86_64.tar.gz

  • 2.卸载centos7自带的Mariadb数据库

    [root@localhost ~]# rpm -qa|grep mariadb  // 查询出来已安装的mariadb  
[root@localhost ~]# rpm -e --nodeps 文件名  // 卸载mariadb，文件名为上述命令查询出来的文件
[root@localhost ~]# rm /etc/my.cnf    //删除etc目录下的my.cnf

  • 3.创建一个用户名为mysql的用户并加入mysql用户组

    [root@localhost ~]# groupadd mysql  
[root@localhost ~]# useradd -g mysql mysql

  • 4.将下载的压缩包放到 /usr/local/ 目录下

  • 5.解压安装包

    [root@localhost local]# tar -zxvf mysql-5.6.40-linux-glibc2.12-x86_64.tar.gz

  • 6.将解压好的文件夹重命名为mysql

    [root@localhost local]# mv 解压出来的文件夹名 mysql

  • 7.在/etc/下新建配置文件my.cnf，并在该文件中添加以下代码：

    [mysql]  
# 设置mysql客户端默认字符集  
default-character-set=utf8   
socket=/var/lib/mysql/mysql.sock  

[mysqld]  
skip-name-resolve  
#设置3306端口  
port = 3306   
socket=/var/lib/mysql/mysql.sock  
# 设置mysql的安装目录  
basedir=/usr/local/mysql  
# 设置mysql数据库的数据的存放目录  
datadir=/usr/local/mysql/data  
# 允许最大连接数  
max_connections=200  
# 服务端使用的字符集默认为8比特编码的latin1字符集  
character-set-server=utf8  
# 创建新表时将使用的默认存储引擎  
default-storage-engine=INNODB  
lower_case_table_name=1  
max_allowed_packet=16M

  • 8.进入安装mysql软件目录

    [root@localhost mysql]# chown -R mysql:mysql ./       修改当前目录拥有着为mysql用户  
[root@localhost mysql]# ./scripts/mysql_install_db --user=mysql --basedir=/usr/local/mysql/ --datadir=/usr/local/mysql/data/           安装数据库

配置MySQL

  • 1.授予my.cnf最大权限

    [root@localhost ~]# chmod 777 /etc/my.cnf

  • 2.设置开机自启动服务控制脚本

    [root@localhost mysql]# cp ./support-files/mysql.server /etc/rc.d/init.d/mysqld     复制启动脚本到资源目录
[root@localhost mysql]# chmod +x /etc/rc.d/init.d/mysqld   增加mysqld服务控制脚本执行权限
[root@localhost mysql]# chkconfig --add mysqld    将mysqld服务加入到系统服务
[root@localhost mysql]# chkconfig --list mysqld    检查mysqld服务是否已经生效,命令输出类似下面的结果:
        mysqld 0:off 1:off 2:on 3:on 4:on 5:on 6:off 
        表明mysqld服务已经生效，在2、3、4、5运行级别随系统启动而自动启动，
        以后可以使用service命令控制mysql的启动和停止
        命令为:service mysqld start和service mysqld stop

  • 3.启动mysqld

    [root@localhost mysql]# service mysqld start

  • 4.将mysql的bin目录加入PATH环境变量，编辑 ~/.bash_profile文件

    [root@localhost mysql]# vim ~/.bash_profile
添加如下信息：
export PATH=$PATH:/usr/local/mysql/bin
之后：
[root@localhost mysql]# source ~/.bash_profile

  • 5.以root账户登录mysql,默认是没有密码的

    [root@localhost mysql]# mysql -uroot -p

  • 6.设置root账户密码为root

    mysql>use mysql;  
mysql>update user set password=password('root') where user='root' and host='localhost';  
mysql>flush privileges;

  • 7.设置远程主机登录，注意下面的your username 和 your password改成你需要设置的用户和密码

    mysql>GRANT ALL PRIVILEGES ON *.* TO 'your username'@'%' IDENTIFIED BY 'your password' WITH  
GRANT OPTION;

  • 8.到此，在Centos 7上安装mysql5.6就完成了

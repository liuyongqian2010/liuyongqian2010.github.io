Nginx转发SSH代理-Stream模块

Nginx转发SSH代理-Stream模块

使用案例

  • 公司的测试主机在内网，外网不能直接访问，可以使用nginx做个tcp代理(使用stream模块)做个端口映射达到可以在本地直接通过外网（可能是已绑定的域名）访问内网测试主机的目的，这里主要做ssh映射。

1.确定内网主机的SSHD服务开启

  • sshd为secure shell的简称，可以通过网络在主机中开机shell的服务默认情况下，sshd服务是开机时自启动的。默认配置文件在/etc/ssh/sshd_config

    #查看服务
systemctl status sshd.service
#重启服务
systemctl restart sshd.service

2.网络防火墙

  • (略)

3.Nginx中stream模块的配置

  • 3.1 stream模块简介

    nginx从1.9.0开始，新增加了一个stream模块，用来实现四层协议的转发、代理或者负载均衡等。使用nginx中的该模块做ssh转发可以连接到外网无法访问的内网主机。
  • 3.2 stream模块编译

    stream模块默认没有编译到nginx， 编译nginx时候需要注意将该模块编译进去：./configure –with-stream即可。

  • 3.3 配置ssh内网转发

    stream {    #stream模块与http模块处于同一级；stream模块编译后需要手动添加；这两个模块可以共存。
    upstream remote01{    
        server 192.168.110.101:22;       #内网服务器101，端口为sshd默认的22
    }
    upstream remote02{
        server 192.168.110.102:22;       #内网服务器102，端口为sshd默认的22
    }

    server {
        listen 2201;                    #监听代理主机的2201端口，通过该端口映射到内网的22端口
        proxy_connect_timeout 1h;
        proxy_timeout 1h;
        proxy_pass remote01;            #转向内网服务器101
    }
    server {
        listen 2202;                    ##监听代理主机的2202端口，通过该端口映射到内网的22端口
        proxy_connect_timeout 1h;
        proxy_timeout 1h;
        proxy_pass remote02;            ##转向内网服务器102
    }
}

注意1：server里面不能够使用location这种语句，否则报错；语法虽与http模块类似，但是有区别的。
注意2：如果与代理主机绑定多个二级域名，在server虚拟机中不可以具体指向二级域名，只需要指明需要监听的代理主机的端口即可，比如上述中的server主机如下设置的话，连接会失败。
server {
        listen ns01.testing.com:2201;   #ns01.testing.com为绑定代理主机的二级域名，通过该域名对应访问内网服务器           
        ......
        proxy_pass remote01;            
    }
    server {
        listen ns02.testing.com:2202;  ##ns02.testing.com为绑定代理主机的二级域名，通过该域名对应访问内网服务器                             
        ......
        proxy_pass remote02;   
    }

  • 3.4 使用shell远程工具通过二级域名访问内网主机时，需要注意填写的端口是代理服务器监听的端口，不是内网服务器sshd默认的22端口。

tomcat使用apr模式运行

tomcat使用apr模式运行

1.安装apr
1.1 在/usr/local解压并进入apr-1.6.5后执行：`./configure --prefix=/usr/local/apr`
1.2 make
1.3 make install
2.安装 apr-iconv
2.1 在/usr/local解压并进入apr-iconv-1.2.2后执行：`./configure --prefix=/usr/local/apr-iconv --with-apr=/usr/local/apr`
2.2 make
2.3 make install
3.安装apr-util
3.0需要安装expat库：yum install -y  expat-devel
3.1 在/usr/local解压并进入apr-util-1.6.1后执行：`./configure --prefix=/usr/local/apr-util --with-apr=/usr/local/apr --with-apr-iconv=/usr/local/apr-iconv/bin/apriconv`
3.2 make
3.3 make install
4. 安装tomcat-native（Tomcat自带tomcat-native.tar.gz，在bin下）
4.1在tomcat/bin下解压并进入tomcat-native-1.2.21-src/native后执行：`./configure --with-apr=/usr/local/apr`
4.2 make
4.3 make install
5.配置APR本地库到系统共享库搜索路径中
vi tomcat/bin/catalina.sh 在虚拟机启动参数JAVA_OPTS中添加java.library.path参数，指定apr库的路径：
# Register custom URL handlers
# Do this here so custom URL handles (specifically 'war:...') can be used in the security policy
JAVA_OPTS="$JAVA_OPTS -Djava.protocol.handler.pkgs=org.apache.catalina.webresources"
JAVA_OPTS="$JAVA_OPTS -Djava.library.path=/usr/local/apr/lib"
6.修改protocol
<Connector  executor="tomcatThreadPool"  port="8080" protocol="org.apache.coyote.http11.Http11AprProtocol" ....../>
7.企业apr生命周期监听器（不使用SSL）
<Listener className="org.apache.catalina.core.AprLifecycleListener" SSLEngine="off" />

批量添加用户，并设置sudo权限

批量添加用户，并设置sudo权限

批量添加用户

  • 1.首先创建一个名称为users.list的文件，将用户名写入进去。

    vi user.list  后按照如下格式添加用户名称即可
zhangsan  lisi wangwu zhaoliu zhangqi wangba chujiu genshi

  • 2.编写批量增加用户的脚本，密码设置为用户名加123

    #!/bin/bash
#
#add users with file called users.list
#
for username in $(more users.list)
do
if [ -n $username ]
then
    useradd -m $username
    echo
    echo $username"123"  | passwd --stdin $username
    echo
    echo "User $username's password is changed!"
else
    echo "The username is null!"
fi
done

赋予普通用户sudo权限

  • 1.root用户为sudo文件增加写权限，默认是读权限：chmod u+w /etc/sudoers

  • 2.打开文件vi /etc/sudoers，在root ALL=(ALL) ALL这一行下面添加普通用户，如下所示

    ## Allow root to run any commands anywhere 
root    ALL=(ALL)     ALL
liuyongqian     ALL=(ALL)   ALL

  • 3.再次取消sudo文件的写权限：chmod u-w /etc/sudoers

  • 4.普通用户sudo时输入的密码是普通用户自己的密码
  • 5.可能需要安装一下sudo，有的系统版本不需要：yum install sudo

Centos中同时部署两个以上的Tomcat

Centos中同时部署两个以上的Tomcat

1.首先配置环境变量

  • 1.1在/usr/local下解压复制两个tomcat，分别重命名为tomcat01、tomcat02.

  • 1.2编辑环境变量

    vi  /etc/profile 在末尾加上以下内容：

##########first tomcat###########
CATALINA01_BASE=/usr/tomcat01
CATALINA01_HOME=/usr/tomcat01
TOMCAT01_HOME=/usr/tomcat01
export CATALINA01_BASE CATALINA01_HOME TOMCAT01_HOME
##########second tomcat##########
CATALINA02_BASE=/usr/tomcat02
CATALINA02_HOME=/usr/tomcat02
TOMCAT02_HOME=/usr/tomcat02
export CATALINA02_BASE CATALINA02_HOME TOMCAT02_HOME

退出并保存，输入 source  /etc/profile  使环境变量起作用

2.修改了另一个Tomcat的文件

  • 2.1进入tomcat02/bin，打开catalina.sh，将所有的CATALINA_BASE改为 CATALINA02_BASE，将所有CATALINA_HOME改为CATALINA02_HOME，并保存。
  • 2.2使用chmod命令保证catalina.sh可执行：chmod u+x catalina.sh

  • 2.3进入tomcat02/conf，打开server.xml，，一共修改三个端口即可

    1.找到<Server port="8005" shutdown="SHUTDOWN">，
改8005为其他未被占用端口；
2.找到<Connector port="8080" protocol="HTTP/1.1"connectionTimeout="20000"  redirectPort="8443" /> ，
改8080为其他未被占用端口；
3.找到<Connector port="8009" protocol="AJP/1.3" redirectPort="8443" />，
改8009为其他未被占用端口。

3.启动tomcat服务

  • 进入tomcat01、tomcat02各自的bin目录，分别执行启动命令：./startup.sh
  • 查看日志：tail -f /usr/tomcat01/logs/catalina.out、
    tail -f /usr/tomcat02/logs/catalina.out

安装部署Mongodb4

安装部署Mongodb4

最新版本

  • 目前最新版本是：mongodb-linux-x86_64-4.0.6

安装步骤

  • 1.解压并重命名mongodb

    将安装包放置在/usr/local/下
解压：
tar -zvxf mongodb-linux-x86_64-4.0.6.tgz
重新命名：
mv mongodb-linux-x86_64-4.0.6 /usr/local/mongodb

  • 2.在/usr/local/mongodb下创建文件夹

    cd /usr/local/mongodb下
mkdir -p ./data/db            #创建数据库文件存放路径
chmod 777 ./data/db
mkdir ./logs                   #创建日志文件
cd logs后
touch mongodb.log

  • 3.在/usr/local/mongodb/bin下新建配置，创建启动文件

    cd /usr/local/mongodb/bin
vi mongodb.conf后增加如下内容：

#数据文件存放目录
dbpath = /usr/local/mongodb/data/db
#日志文件存放目录
logpath = /usr/local/mongodb/logs/mongodb.log
#端口
port = 27017
#以守护程序的方式启用，即在后台运行
fork = true

logappend=true
maxConns=10000
bind_ip=0.0.0.0

配置环境变量

  • 1.执行

    vi /etc/profile后增加如下内容：

MONGODB_HOME=/usr/local/mongodb
PATH=$PATH:$MONGODB_HOME/bin
export MONGODB_HOME PATH

  • 2.使配置文件生效执行

    source /etc/profile

启动关闭服务

  • 在/usr/local/mongodb/bin下执行启动命令

    cd /usr/local/mongodb/bin
mongod -f mongodb.conf   或 
./mongod -f mongodb.conf

  • 执行访问命令

    ./mongo

  • 执行关闭命令

    mongod -f ./mongodb.conf --shutdown  或 
./mongod -f ./mongodb.conf --shutdown

开启端口

  • 开启端口

    firewall-cmd --zone=public --add-port=27017/tcp --permanent

  • 查看端口

    firewall-cmd --permanent --query-port=27017/tcp

安装部署Nginx

安装部署Nginx

依赖安装包

  • nginx编译依赖gcc环境

    yum install gcc-c++

  • nginx的http模块使用pcre来解析正则表达式

    yum install -y pcre pcre-devel

  • nginx使用zlib对http包的内容进行gzip

    yum install -y zlib zlib-devel

  • nginx需要一个强大的安全套接字层密码库

    yum install -y openssl openssl-devel

目前版本

  • 目前版本：nginx-1.14.2

安装步骤

  • 1.将安装包放置在/usr/local下并解压：

    tar -zvxf nginx-1.14.2.tar.gz

  • 2.进入解压后的文件夹内执行如下配置编译命令：

    cd nginx-1.14.2
如果使用Nginx1.9之后的stream模块的话，执行：
./configure --with-stream
不需要的话，执行：
./configure

  • 3.编译安装，分别执行：

    make
make install

启动停止nginx

  • 查询

    ps aux|grep nginx

  • 启动

    cd /usr/local/nginx/sbin/后执行：
./nginx

  • 待nginx进程处理任务完毕进行停止

    ./nginx -s quit

  • 先查出nginx进程id再使用kill命令强制杀掉进程

    ./nginx -s stop

  • 如果nginx.conf 修改后需要重启nginx使用

    ./nginx -s reload

安装部署mysql5.7

安装部署mysql5.7

删除centos7.4自带的mysql分支mariadb

  • 查看删除

    查看：rpm -qa | grep mariadb
删除1：yum -y remove mari*
删除2：rpm -e --nodeps ****
find / -name mysql
rm -rf *****
rm -f /etc/my.cnf
rm -rf /etc/init.d/mysql.server
userdel mysql         #删除
id mysql              #查看

目前版本

  • 目前安装版本是mysql-5.7.25-linux-glibc2.12-x86_64

解压安装包

  • 解压并重命名为mysql

    将安装包放置于/usr/local/下并解压：
tar -zvxf mysql-5.7.25-linux-glibc2.12-x86_64.tar.gz
重新命名：
mv mysql-5.7.25-linux-glibc2.12-x86_64 mysql

安装依赖包

  • 执行

    yum -y install libaio

初始化环境并配置文件

  • 1.创建mysql用户和组

    groupadd mysql
useradd -r -g mysql -s /bin/false mysql
chown -R mysql.mysql /usr/local/mysql

  • 2.创建data文件夹

    在/usr/local/mysql下执行命令：
cd /usr/local/mysql
mkdir data

  • 3.修改my.cnf

    vi /etc/my.cnf后写入文件内容如下：

[mysqld]
port=3306
character-set-server=utf8
basedir=/usr/local/mysql
datadir=/usr/local/mysql/data
#innodb_buffer_pool_size=128M
explicit_defaults_for_timestamp=true
[mysqld_safe]
log-error=/usr/local/mysql/data/error.log
pid-file=/usr/local/mysql/data/mysql.pid
tmpdir = /tmp
[client]
default-character-set=utf8
[mysql]
default-character-set=utf8

  • 4.将mysql放入系统服务，并修改对应文件

    cp /usr/local/mysql/support-files/mysql.server /etc/init.d/mysqld
vi /etc/init.d/mysqld  后修改文件路径:(3处)

# If you change base dir, you must also change datadir. These may get
# overwritten by settings in the MySQL configuration files.

basedir=/usr/local/mysql
datadir=/usr/local/mysql/data

# Default value, in seconds, afterwhich the script should timeout waiting
# for server start. 
# Value here is overriden by value in my.cnf. 
# 0 means don't wait at all
# Negative numbers mean to wait indefinitely
service_startup_timeout=900

# Lock directory for RedHat / SuSE.
lockdir='/var/lock/subsys'
lock_file_path="$lockdir/mysql"

# The following variables are only set for letting mysql.server find things.

# Set some defaults
mysqld_pid_file_path=/usr/local/mysql/data/mysql.pid

  • 5.初始化mysql

    cd /usr/local/mysql后执行：
./bin/mysqld --initialize --user=mysql --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data

修改root密码并设置远程连接

  • 启动mysql：service mysqld start后依次执行

    cd /usr/local/mysql后执行：./bin/mysql -u root -p
输入初始化时生成的默认密码
修改密码：SET PASSWORD FOR 'root'@localhost='123456';

允许远程连接：
use mysql;
update user set host='%' where user='root' limit 1;
flush privileges;

配置环境变量

  • 依次执行

    vi /etc/profile后增加如下内容：

MYSQL_HOME=/usr/local/mysql
PATH=$PATH:$MYSQL_HOME/bin
export MYSQL_HOME PATH

  • 使配置文件生效执行

    source /etc/profile

添加需要监听的端口

  • 执行

    /sbin/iptables -I INPUT -p tcp --dport 3306 -j ACCEPT

参考Centos7.5安装mysql5.7.23-二进制包方式安装-完美安装,一步到位

参考Centos7.4 安装MySQL 5.7.21 (通用二进制包)
